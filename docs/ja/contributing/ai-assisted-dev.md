# AI支援開発

N.E.K.O. はリポジトリのルートに `.agent/` ディレクトリを提供しています。この規約をサポートするAIコーディングアシスタント——**Claude Code**、**Cursor**、**Windsurf** など——は、定義されたルールとスキルセットを自動的に読み込み、プロジェクトのコーディング規約に従います。

## ディレクトリ構造

```
.agent/
├── rules/
│   └── neko-guide.md        # コア開発ルール（常に読み込み）
└── skills/                  # タスク別スキルセット
    ├── i18n/
    ├── frontend-design/
    ├── vrm-physics/
    ├── tts-error-reporting/
    └── ...                  # 合計約20スキル
```

## ルール

`.agent/rules/neko-guide.md` は `trigger: always_on` としてマークされており、互換性のあるAIエージェントはセッション開始時に必ず読み込みます。カバーする内容：

- **i18n**：翻訳文字列を変更する際は、6つのlocaleファイルすべてを同時に更新する必要があります。
- **`uv run`**：システムPythonではなく、常に `uv run` を使用してPythonを実行します。
- **コードの対称性**：並列のプロバイダーは対称的な構造を維持する必要があります。
- **coreレイヤーの純粋性**：`core.py` にプロバイダー固有のロジックを含めてはいけません。
- **数字サフィックスの禁止**：共有ロジックはメソッドに抽出します。
- **Electron vs. 開発モード**：フロントエンドの変更は両方の実行環境で動作する必要があります。

これらのルールは `CLAUDE.md`（Claude Code専用）およびプロジェクトの[コードスタイル](/ja/contributing/code-style)ガイドと一致しています。

## スキル

`.agent/skills/` 配下の各サブディレクトリには `SKILL.md` が含まれ、以下を定義しています：

- **トリガー条件** — 例：i18nファイルの編集時、VRM物理の作業時、フロントエンドUIの構築時。
- **ドメイン知識** — 関連するリファレンス、パターン、注意点。
- **ステップバイステップのワークフロー** — その領域の一般的なタスクの処理手順。

スキルは、現在のタスクがトリガー条件に一致した場合にAIエージェントによってオンデマンドで読み込まれます。

## コントリビューターの方へ

AIコーディングアシスタントを使用してN.E.K.O.に貢献する場合、`.agent/` ディレクトリにより、AIは自動的に：

1. プロジェクトのコードスタイルとアーキテクチャルールに従います。
2. 翻訳文字列の変更時に6つのi18n localeすべてを同時に更新します。
3. Python実行には `uv run` を使用します。
4. Electron と開発モードの区別を尊重します。

追加の設定は不要です——プロジェクトを開いてコーディングを始めるだけです。

## その他のAIツール

`.agent/` の規約は新興の標準です。お使いのAIツールがネイティブにサポートしていない場合でも、`.agent/rules/neko-guide.md` をシステムプロンプトやプロジェクトコンテキストファイルとして手動で読み込むことができます。内容はプレーンなMarkdownであり、特定のツールに依存しません。
