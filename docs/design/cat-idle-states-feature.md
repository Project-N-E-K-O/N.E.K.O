# 猫娘空闲状态分层 — 功能链路

> 详细代码现状见 [cat-idle-states-reference.md](./cat-idle-states-reference.md)

## 背景

当前代码里，"请她离开"后会隐藏模型，并显示一个回来入口；当前该入口已替换为 `CAT1` 形态，首页聊天框仍可折叠为最小化球。

优化目标：让猫娘在不活跃时也保持轻量、可见的陪伴感。

---

## 一、"请她离开"的变化

### 当前流程

```text
点击"请她离开"
  ├─ 设置 _goodbyeClicked = true（三个 Manager）
  ├─ 关闭所有弹窗、重置浮动按钮
  ├─ 模型锁定、canvas 隐藏（1100ms 后 visibility/display none）
  ├─ 浮动按钮容器隐藏
  ├─ 侧栏隐藏
  ├─ 对话框折叠到最小化/移动端折叠
  ├─ 显示回来入口
  └─ 触发 resetSessionButton.click()
       ├─ 输入区域隐藏，所有按钮禁用
       ├─ WebSocket end_session
       ├─ 停止 proactive 调度
       └─ returnSessionButton 启用
```

### 当前代码口径

在上述流程基础上：

1. 原本的回来入口，当前已替换为 `CAT1 / CAT2 / CAT3` 三段表现
2. 这三段形态只代表 goodbye 之后的视觉层，不是新的业务状态
3. 其他 goodbye / return 语义保持原链路不变

---

## 二、空闲退化

当前代码里，这不是独立于"请她离开"的全局状态机，而是：无阻断时自动复用现有 goodbye，随后只切换 `CAT1 / CAT2 / CAT3` 表现。

### 流程

```text
用户最后一次有效交互时间开始计时
    │
    20 分钟无交互且无阻断
    │（当前代码联调值：5s）
    ├─ 自动触发现有 goodbye
    └─ 回来入口显示 CAT1
            │
            ├─ 累计到 30 分钟（当前代码联调值：10s）→ CAT2
            └─ 累计到 40 分钟（当前代码联调值：15s）→ CAT3
```

补充：

- `CAT2 / CAT3` 下，首页最小化聊天球会停靠到猫左侧
- `/chat` 独立窗口不运行 auto-goodbye 控制器，只回传真实交互

### 恢复

当前代码**不做任意交互自动唤醒**。

- 活跃态下的点击、触摸、按键、滚轮、语音等真实交互会重置空闲计时
- 已进入 `CAT1 / CAT2 / CAT3` 后，需要点击现有回来入口恢复
- 恢复继续走当前 return 链，不切到 `returnSessionButton -> start_session`

## 二点五、与当前代码的映射

| 概念 | 对应现状 |
|------|---------|
| 模型显示区域 | `#live2d-container` / `#vrm-container` / `#mmd-container` |
| 对话框 | 首页真正可见的是 React chat host；旧 `#chat-container` 仍保留兼容链路 |
| 对话框小球 | React chat host 的 50×50 最小化球 |
| 回来入口 | 各 Manager 的 return-button-container / `*-return-click` |
| `_goodbyeClicked` | 三个 Manager 各自持有，全局通过 `isGoodbyeActive()` 检查 |
| 空闲计时 | 已存在于 `static/app-auto-goodbye.js`，当前联调阈值为 `5s / 10s / 15s` |

## 三、点击形态

当前三种猫形象都已有默认资源与点击态资源，点击后仍以"回来"为主语义。

| 形态 | 默认态 | 被点击态 |
|------|--------|---------|
| **猫形象一** | `CAT1` GIF | 点击后继续走当前回来链 |
| **猫形象二（打盹）** | `CAT2` GIF | 点击后继续走当前回来链 |
| **猫形象三（睡觉）** | `CAT3` GIF | 点击后继续走当前回来链 |

当前 hover / click 反馈的代码口径：

1. 鼠标进入时切到当前 tier 对应的 `*-click.gif`。
2. 鼠标离开时不会立刻切回默认态，而是读取该 click GIF 自身的帧延迟总和，等一轮播放完成后再恢复默认 GIF。
3. 反复鼠标进入 / 移出同一个 tier 时，不会重复设置同一个 `src`，因此不会让 GIF 一直从第一帧重新播放。
4. tier 切换会清理旧 hover 播放状态，避免跨 CAT 阶段串图。

更细的动画、音效和正式资源仍可后续替换，但不应改变回来语义。

## 四、资源状态

| 用途 | 当前代码 | 备注 |
|------|---------|------|
| 猫形象一 | GIF 资源 | goodbye / auto-goodbye 后显示 |
| 猫形象二 | GIF 资源 | 30min 目标档位；当前联调值对应 10s |
| 猫形象三 | GIF 资源 | 40min 目标档位；当前联调值对应 15s |
| 对话框球（猫形象二/三） | CSS + React chat idle-dock | 仅在 CAT2 / CAT3 下停靠到猫左侧 |

### 给美术组的出图要求

按美术组正式交付规格，三种状态和各自点击态**统一按 GIF 输出**。

说明：

1. 当前工程接入口径已经统一为 GIF。
2. 本节面向美术出图，正式资源继续按 GIF 规划。
3. 前端后续替换正式资源时，应保持现有文件名和 GIF 格式，避免再引入 png/gif 混用。

| 状态 | 默认资源 | 点击态资源 | 美术交付格式 |
|------|---------|-----------|-------------|
| `CAT1` | `cat-idle-cat1.gif` | `cat-idle-cat1-click.gif` | GIF + GIF |
| `CAT2` | `cat-idle-cat2.gif` | `cat-idle-cat2-click.gif` | GIF + GIF |
| `CAT3` | `cat-idle-cat3.gif` | `cat-idle-cat3-click.gif` | GIF + GIF |

统一要求：

1. 建议透明背景，四周保留安全留白，不要做满画布。
2. 当前展示容器是**正方形**，桌面端约 `96px ~ 122px`，移动端约 `88px ~ 108px`，并且使用 `object-fit: contain`；因此主视觉应放在正方形安全区中央。
3. 画面重心建议略偏下，让猫落点更稳，避免顶部耳朵或尾巴在小尺寸下显得飘。
4. 不要把“请她回来”等文字直接画进正式资源里，后续应以纯美术图为主。
5. 点击态不是新动作语义，仍然表示“点她回来”；因此点击态建议只做**轻微确认反馈**，不要做夸张变身或完全换构图。
6. 既然统一为 GIF，默认态和点击态都建议做成**短循环、可无缝衔接**的动图；点击态可以更明显，但仍应控制在轻反馈范围内。

分状态建议：

| 状态 | 目标氛围 | 默认态建议 | 点击态建议 |
|------|---------|-----------|-----------|
| `CAT1` | 刚离开但还在陪伴 | 清醒、看着用户、可爱待机 | 轻微靠近、抬爪、眨眼、耳朵/尾巴有回应感 |
| `CAT2` | 打盹、较弱活跃感 | 团起来或趴着小憩，放松但未睡沉 | 微微睁眼、耳朵轻动、短暂被叫醒感 |
| `CAT3` | 睡着、最弱活跃感 | 明确睡着状态，动作更缓、更弱 | 点击态建议做“被碰一下”的短反馈，如皱眉、动耳、尾巴轻抖 |

构图建议：

1. 三个状态尽量保持**朝向一致、主体尺度接近、落点一致**，避免切换 tier 时位置跳动。
2. 默认态和点击态尽量共用同一套轮廓边界，避免 hover / click 时画面突然膨胀或偏移。
3. `CAT2 / CAT3` 旁边未来会停靠聊天球，因此猫的左侧轮廓建议不要过度外扩，给左侧球预留视觉呼吸空间。
4. 三档 GIF 都建议小循环、低频率、低振幅；`CAT1` 可以最活，`CAT2` 次之，`CAT3` 最弱，避免常驻状态下过分吸睛。

## 五、边界场景

| 场景 | 预期 |
|------|------|
| 活跃态下长时间无交互且无阻断 | 自动复用现有 goodbye，先进入 CAT1 |
| 已处于 goodbye 后继续闲置 | 退化到 CAT2 / CAT3；CAT2/3 下聊天球停靠到猫左侧 |
| 已处于 CAT1 / CAT2 / CAT3 时只是触摸/按键/滚轮 | 不自动恢复，也不重置当前 tier |
| 已处于 CAT2 / CAT3 时拖拽猫形象 | 拖拽和松手都不应让状态退回 CAT1 |
| 已处于 CAT1 / CAT2 / CAT3 时 hover 猫形象 | 播放对应点击态 GIF；鼠标离开后等该 GIF 一轮播放完成再恢复 |
| 点击 CAT1 / CAT2 / CAT3 回来入口 | 继续走当前 return 链恢复前台 |
| 关闭重开 | 状态不持久化，回到活跃态 |
