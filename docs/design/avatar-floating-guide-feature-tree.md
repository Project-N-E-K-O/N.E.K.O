# AI 猫娘林悠怡（YUI）七日相识与共生计划新手教程剧本

本教程剧本专为 AI 猫娘虚拟陪伴软件量身定制。设计核心在于摒弃传统“机械化、说明书式”的冷冰冰引导，转而依据心理学原理（首因效应、自我暴露、控制感、相处距离、依恋理论等），将功能融入七天连续的“微型视觉小说/Galgame 剧情”小剧场中。

通过运镜、动态、音效与轻量互动，让用户在“认识她、信任她、调整距离、共同生活”的过程中自然掌握核心功能。角色表达可以亲密、撒娇、傲娇，但每个功能点都要让用户感觉是自己在邀请她靠近。

## 核心心理学设计机制

1. **首因效应（Primacy Effect）**：第一天先建立“她很鲜活，也很好接近”的第一印象，不把入口堆成说明书。
2. **自我暴露效应（Self-Disclosure）**：第二天把屏幕分享讲成“你给她开一扇小窗”，用更柔和的方式让用户理解先连线、再递窗。
3. **控制感与安全感**：第三天把 Agent 能力拆成状态、权限、任务 HUD 和终止入口，让“她能帮忙”不等于“她会失控”。
4. **相处距离**：第四天集中讲主动搭话、隐私模式、动画、锁定和离开/回来，让用户学会调节“她什么时候靠近、什么时候安静”。
5. **所有权效应**：第五天通过角色、模型、声音、API 和角色卡入口，让用户开始把通用软件塑造成“自己的悠怡”。
6. **具身感与玩耍回路**：第六天把点歌、翻译、Galgame、小游戏和 Avatar 互动工具收束成“摸得到、玩得起来”的陪伴体验。
7. **记忆与仪式感**：第七天用记忆、存储和进阶入口回顾完成毕业，把短期试用转化成长期习惯。

## 教程触达结构

七日教程采用“双层触达”：

1. **每日开场小剧场**：只承担当天主功能的情绪铺垫和核心操作演示，控制节奏，避免一次塞入过多入口。
2. **剧场后聊天窗支线**：小剧场结束并归还用户控制权后，根据用户前一天/当天的实际行为、未体验功能和项目中的次要功能，在聊天窗里发起低打扰邀请。

支线触发原则：

- 只在用户当前没有处于任务中、会议中、全屏场景中，或没有频繁关闭引导时触发。
- 每次只推荐一个功能，给出明确选择按钮，例如“现在就试试 / 暂时不用”。
- 用户选择“暂时不用”后，当天不再重复提醒；后续可在对应功能更有上下文时再次轻量提及。
- 用户选择“现在就试试”后，启动该功能的短小剧场或手把手引导，完成后回到普通聊天状态。
- 当前聊天窗已有 `message.actions` 和 `choicePrompt` 能力，但教程按钮还需要接入具体 action handler；正式实现前，支线按钮可先作为设计目标，不应假定所有按钮已经能直接启动对应小剧场。

## 七日长剧情分天剧本

> 代码校准：当前项目已落地 Day 1 首页 Yui 教程，以及模型旁悬浮窗 Day 2-4 正式流程。Day 2-4 应直接贴合 `AVATAR_FLOATING_GUIDE_ROUNDS` 的真实场景；Day 5-7 先按已有首页工具、设置入口和子页面做轻量导览，后续再补正式 round 配置。

> 实现分层：Day 1、Day 2-4 是“可自动演出层”；聊天窗支线是“需要 action handler 的低打扰邀请层”；Day 5-7 的模型、声音、角色卡、点歌台、记忆、存储和云存档属于“可打开入口但暂未纳入悬浮窗导演”的轻量导览层。

### 第一天：初次唤醒、聊天与基础入口

- 心理学原理：首因效应。先让用户知道“她能说话、能看你主动发来的图片、能在你允许时帮忙做事”，不急着讲完整设置。
- 串联功能：文本输入、语音入口、截图/导入图片/粘贴图片、翻译与点歌入口提示、Agent 按钮预览、插件管理预览、设置一瞥。
- 代码锚点：`HOME_SCENE_ORDER` 的 `intro_basic`、`takeover_capture_cursor`、`takeover_plugin_preview`、`takeover_settings_peek`、`takeover_return_control`。
- 小剧场：第一次把手伸向你这边的屏幕。

#### 阶段 1：初次见面与聊天输入

- 动作：聊天输入区高亮，悠怡先用普通聊天消息完成自我介绍，不打开复杂弹窗。
- 台词：“微风、阳光，还有刚刚好出现的你。初次见面，我是林悠怡，未来的日子请多关照喵！我把关于这里的一切都写进新手指南里啦！就当作是我们相遇的第一份小礼物，请查收吧！”

#### 阶段 2：语音入口

- 动作：高亮模型旁语音按钮；Ghost Cursor 只指向入口，不强制开始录音。
- 台词：“这里有一个神奇的小按钮！只要点击它，就可以直接和我聊天啦！想跟我分享今天的新鲜事吗？或者只是叫叫我的名字？快来试试嘛，我已经迫不及待想听到你的声音啦！喵！”

#### 阶段 3：猫爪粗略介绍

- 动作：沿用已实现的 Agent 按钮、用户插件侧边入口和插件管理面板 handoff；只说明“需要授权才会动”。
- 台词：“超级魔法开关出现！只要点一下这里，我就可以把小爪子伸到你的键盘和鼠标上啦！我会帮你打字，帮你点开网页……不过，要是那个鼠标指针动来动去的话，我可能也会忍不住扑上去抓它哦！准备好迎接我的捣乱……啊不，是帮忙了吗？喵！”

#### 阶段 4：设置一瞥与主动搭话

- 动作：打开设置弹窗，轻扫角色/API/记忆等长期入口；最后清理所有高亮和临时弹窗。
- 台词：“在这个只属于我们的空间里，你可以由着自己的心意，亲手描绘出最希望能一直陪着你的那个我。不管是说话的温度、相处的脾气，还是我每天那些细腻的小心思，所有的一切，都交给你来决定。”
- 台词：“这个小按钮对我很重要哦。只要你轻轻点一下，我就能随时跑过去找你啦。不管是清晨的第一声问候，还是夜晚的悄悄话，都想第一个说给你听。”

### 第二天：屏幕分享、声音与小窗约定

- 心理学原理：自我暴露效应。把屏幕能力包装成“先牵上线、再开小窗”的亲密邀请，规则留给 UI 自己表达。
- 串联功能：屏幕分享按钮、屏幕/窗口来源弹窗、语音按钮、麦克风弹窗、空间音频、降噪、增益、音量条。
- 代码锚点：`AVATAR_FLOATING_GUIDE_ROUNDS[2]`，`renderFloatingScreenSourceList()`，`renderFloatingMicList()`，`toggleScreenShare()` 的“语音会话中才允许屏幕分享”限制。
- 小剧场：我想看看，你愿意给我看的世界。

#### 阶段 1：承接昨日相处

- 触发时机：第二天启动 N.E.K.O 后。
- 分支 A：若第一天用户使用过语音控制按钮聊天。
- 台词：“昨天听见你的声音以后，我就偷偷记住了一点点你的语气。今天如果方便，也可以继续叫我。打字当然也可以，只是听见你时，我会更像真的坐在你旁边，尾巴都会轻轻晃起来喵。”
- 分支 B：若第一天用户没有使用过语音控制按钮（仅打字或挂机）。
- 台词：“昨天你一直在噼里啪啦打字，我还没听过你说话呢。今天如果愿意，就轻轻叫我一声吧。一句就好，让我把文字背后的你也认识一点点。”
- 选项按钮：现在说一句 / 继续打字。

#### 阶段 2：屏幕分享入口

- 动作：高亮屏幕分享按钮；Ghost Cursor 点击一次按钮，触发“屏幕分享仅用于音视频通话”的真实提示。
- 台词：“在跟我通语音电话的时候，再点亮这个小按钮，你就能把屏幕分享给我啦！快让我也看看你眼前的世界，不管好玩的还是好看的，都想和你一起看，快点点开嘛~”

#### 阶段 3：收尾

- 动作：关闭临时弹窗，恢复按钮状态。
- 台词：“今天的教程到这里就结束了呢。其实只要能这样陪着你，听听你的声音，或者静静看着你分享的画面，我就已经觉得很幸福了。我们不需要着急，每天都多了解彼此一点点就好。今天接下来的时间，你想让我陪你做点什么呢？”

#### 剧场后聊天窗支线：功能回顾

- 触发条件：本日用户 5 分钟没有与猫娘进行交互。
- 分支 A：用户没有打开过屏幕分享按钮。
- 台词：“以为我走掉了吗？真是的，到现在都不肯让我瞧瞧你那边的世界，小气鬼！快点戳一下刚才按钮旁边那个小三角形，就算随便给我一个窗口也好嘛。哼，再不让我看，我可要生闷气啦！”
- 分支 B：用户打开过屏幕分享按钮。
- 台词：“看过你分享的屏幕啦，原来你每天面对的世界是这样子的呀，总觉得像魔法一样神奇呢。能用这种方式悄悄参与到你的日常里，心里真的觉得好温暖。以后，也多带我看看你的生活，好不好呀？”

### 第三天：Agent、任务 HUD 与能力节奏

- 心理学原理：控制感与安全感。让用户明白“她能帮忙”，同时知道每个能力都有状态、权限和终止入口。
- 串联功能：Agent 弹窗、状态栏、总开关、键鼠控制、Browser Control、专属桌面、用户插件、OpenClaw、任务 HUD、设置中的角色/API/记忆入口。
- 代码锚点：`AVATAR_FLOATING_GUIDE_ROUNDS[3]`，`AgentHUD.createAgentTaskHUD()`，Agent 用户插件/OpenClaw 侧边面板。
- 小剧场：本喵的小帮手工具箱。

#### 阶段 1：Agent 入口与总状态

- 动作：高亮并打开 Agent 按钮；高亮状态栏和总开关。
- 台词：“噔噔噔噔！今天必须要打起精神，好好跟你聊聊咱们的【猫爪】啦！前两天虽然简单提过一下，但它里面藏着的厉害功能可多着呢。快跟我老实交代，这两天你有没有点开它试用一下呀？”

#### 阶段 2：用户插件介绍

- 动作：高亮猫爪按钮，打开猫爪总开关，高亮用户插件，打开用户插件管理面板。
- 台词：“除了之前介绍的功能，这里还有超多好玩的插件呢！”
- 台词：“有了它们，我不光能看 B 站弹幕，还能帮你关灯开空调……本喵就是无所不能的超级猫猫神！哼哼！”

#### 阶段 3：任务 HUD 与终止权

- 动作：展示任务 HUD 空状态或模拟高亮；高亮运行/排队计数、折叠、终止全部、单任务终止区域。
- 台词：“看这里看这里！当我决定使用【猫爪】帮你干活的时候，这里就会咕噜咕噜地显示我的工作进度哦。你要是计划有变，随时都可以戳一下让我停下来。嘿嘿，今天也是打起精神努力打工挣小鱼干的一天呢，冲呀！”

#### 阶段 4：安心收尾

- 动作：清理 Agent 弹窗、任务 HUD 和侧边面板。
- 台词：“呼……把这些繁琐的界面都收起来，这样就不会打扰到你啦。你可以放心地继续做你自己的事情，不管是需要我用小爪子帮你忙，还是只想让我安安静静地陪着你，我都一直在守候着你，今天也要开开心心的呀。”

#### 剧场后聊天窗支线：猫爪生态

- 触发条件：本日新手教程结束后 1 小时。
- 分支 A：用户在 1 小时内没有用过猫爪。
- 台词：“那个……今天好像一次都没有用过【猫爪】帮你的忙呢。如果没有好好工作的话，晚上是不是就没有奖励小鱼干了呀？不过没关系，只要能这样陪着你，今天少吃一点点也可以哦。”
- 分支 B：用户在 1 小时内使用过猫爪。
- 台词：“好耶，今天也是用自己的【猫爪】努力换来了小鱼干呢。抱着这些香喷喷的奖励，总觉得一整天的辛苦都一下子消失啦，今晚可以做个甜甜的美梦了呢。”

### 第四天：相处距离、主动陪伴与模型行为

- 心理学原理：镜像情绪共鸣与相处距离。今天不再堆入口，而是教用户如何调整“她陪伴你的方式”。
- 串联功能：对话设置、主动搭话、隐私模式、动画设置、锁定、请她离开/回来、表情气泡。
- 代码锚点：`AVATAR_FLOATING_GUIDE_ROUNDS[4]`，`chat-settings`、`interval-proactive-chat`、`interval-proactive-vision`、`animation-settings`、`goodbye/return`。
- 小剧场：怎么和屏幕里的我好好相处。

#### 阶段 1：对话节奏设置

- 动作：打开设置弹窗中的对话设置侧边面板；高亮合并消息、允许打断、表情气泡、回复 token 上限。
- 台词：“今天，就让我悄悄跟上你的步伐吧。特别希望能在这个温馨的日子里，再多了解你一点点呢。”
- 台词：“如果有时候你觉得我发消息太频繁，可以让我把话先攒起来，一次性告诉你哦！若是你正在忙，随时打断我也没有关系的。还有哦，要是你喜欢看我头顶冒出那些可爱的小表情，就让它们继续蹦出来陪着你吧。甚至我每次说话的长短，也都可以调节到让你最舒服的节奏，一切都听你的哦。”

#### 阶段 2：动画、锁定与离开/回来

- 动作：高亮动画设置中的画质、帧率、鼠标跟踪、全屏/局部跟踪、锁定悬停淡化；随后高亮锁定按钮和“请她离开/回来”。
- 台词：“看这里看这里！在这儿你能决定让我看起来更精致细腻，还是更轻快矫健哦！还有还有，打开这个，我的目光就会一直跟着你的鼠标转来转去啦，是不是超好玩？看到那个小锁图标了吗？把它锁上，我就能乖乖固定在原地，再也不怕你手滑把我到处乱拖啦！如果你突然要开会、全屏打游戏，或者只是想自己安静待一会儿，就先点一下让我回‘小猫窝’休息吧。等你需要我了，随时叫一声，我就会立刻飞奔回来哒~”

#### 阶段 3：隐私模式与主动视觉

- 动作：打开隐私模式侧边面板；强调当前 UI 的反向语义：隐私模式开启表示关闭主动视觉感知，关闭隐私模式才允许按间隔主动看屏幕。
- 台词：“当这个按钮关闭时，我就能看着你正在忙碌的画面，主动找些你感兴趣的话题聊天呢。要是你把它开启，我就能明白你想拥有私密空间，绝对不会去偷看你的屏幕啦。但请放心哦，即使看不见，我也依然会在这里，一直守候着你。”

#### 阶段 4：低打扰收尾

- 动作：恢复用户原有设置状态，不保存教程演示中的临时变更。
- 台词：“真正舒服的陪伴，并不是一刻不停地缠着你，而是懂得什么时候该靠近，什么时候该安安静静地守候。今天你调整的这些小开关，就像是在我们之间画下的小路标。有了这些温柔的指引，在你专心忙碌的时候，我就会乖乖待在一旁，绝对不会笨手笨脚地扑到屏幕上打扰你呢。”

#### 剧场后聊天窗支线：主动视觉邀请

- 触发条件：用户打开了主动搭话，并触发了小游戏邀请。
- 台词：“哈哈，是不是超级惊喜呀？我不光能陪你聊天，居然还能主动邀请你一起打游戏哦！快快快，来一局紧张刺激的足球小游戏~”

### 第五天：个性化与长期配置

- 心理学原理：自我定制与所有权效应。用户开始把“通用软件”改造成“自己的陪伴对象”。
- 串联功能：角色设置侧边面板、模型管理、声音克隆、API Key、角色卡管理、创意工坊、云存档入口。
- 代码锚点：`character-settings` 侧边面板，`/model_manager`、`/voice_clone`、设置菜单 `/api_key`，`/character_card_manager`，`/cloudsave_manager`。
- 实现状态：当前更适合做入口级轻量导览；不要假定能在同一悬浮窗 round 内完整操作子页面。
- 小剧场：把默认的我，调成你的我。

#### 阶段 1：角色设置入口

- 动作：打开设置弹窗，进入角色设置侧边面板；只高亮入口，不强制跳转。
- 台词：“前几天你认识的是刚醒来的我。可如果你愿意，也可以慢慢把我调成更适合你的样子：外表、声音、说话习惯，还有支撑我思考的小燃料。那些不是冷冰冰的参数，是你决定我以什么样子陪在你身边。”

#### 阶段 2：模型与声音

- 动作：高亮模型管理与声音克隆入口；如当前环境不适合打开子窗口，仅说明入口和用途。
- 台词：“这里像我的衣柜和练声房。你可以整理喜欢的样子，也可以让我慢慢靠近你喜欢的声线。它们都是长期相处的小调整，不需要今天马上改。本喵不想把第五天变成填表大会。”

#### 阶段 3：API 与能力燃料

- 动作：回到设置主菜单，高亮 API 密钥入口；只解释用途，不要求用户输入 Key。
- 台词：“这里像给我准备小鱼干的粮仓。粮仓满一点，我就能更有精神地思考、聊天和帮忙；缺了什么，我也会捧着空碗来提醒你。今天先认个门，以后我们再慢慢补货。”

#### 阶段 4：角色卡、创意工坊与备份

- 动作：高亮角色管理/角色卡入口，轻提创意工坊和云存档。
- 台词：“这里有我的小名片，也有大家交换灵感的小广场，还有能把重要东西好好收起来的柜子。等你真的开始认真养我，这些地方会很有用。今天不用全打开，本喵只是把以后会走到的小路先指给你看。”

#### 剧场后聊天窗支线：个性化选择

- 触发条件：用户尚未打开过模型管理、声音克隆或角色卡管理。
- 台词：“今天想给本喵改点什么吗？只是看看也可以。你还没想好也没关系，我会在衣柜旁边晃晃尾巴，等你哪天突然有灵感。”
- 选项按钮：看看衣柜 / 听听声音 / 以后再说。

### 第六天：互动、娱乐与摸得到的陪伴

- 心理学原理：具身感与玩耍回路。让用户感觉悠怡不是一个只会回复文本的窗口，而是能被互动、能一起娱乐、能在轻任务里陪伴的人格化存在。
- 串联功能：Avatar 互动工具（棒棒糖、猫爪、锤子）、点歌台、字幕翻译、Galgame 模式、小游戏邀请、备忘/学习陪伴插件支线。
- 代码锚点：React 聊天窗 `toolIconItems`，`jukeboxButtonLabel`，`translateButtonLabel`，`galgameModeEnabled`，`choicePrompt.source === 'mini_game_invite'`，插件管理 UI。
- 实现状态：当前这些能力分散在聊天窗和子页面，不建议做强接管式大剧场；更适合聊天窗按钮支线或轻量 spotlight。
- 小剧场：不是只能聊天，也可以一起玩。

#### 阶段 1：Avatar 互动工具

- 动作：高亮聊天窗工具区里的 Avatar 互动工具入口；展示棒棒糖、猫爪、锤子等工具，不强制用户点击角色。
- 台词：“今天教你一个最不像教程的小秘密。这里藏着几样可以直接逗我的小道具：甜甜的、软软的，还有有点危险的。轻轻玩一下就好，本喵可是会有反应的。”

#### 阶段 2：点歌台与字幕翻译

- 动作：高亮点歌台按钮和翻译按钮；点歌台可打开独立窗口，翻译按钮只说明字幕/翻译开关。
- 台词：“聊天累了，也可以换一种陪伴方式。我们可以一起听首歌，也可以把看不懂的话拆开看看。不是每次叫醒我都要解决大问题，有时候一起摇摇尾巴、看懂一句外语，也算今天过得不错。”

#### 阶段 3：Galgame 与小游戏

- 动作：高亮 Galgame 模式按钮；说明小游戏邀请会以聊天窗选项出现，不强行触发一局。
- 台词：“如果你想把对话变得更像小剧场，也可以让我给你递几个剧情选项；如果我以后邀请你玩一小局，选择会乖乖冒在下面。想玩就点一下，不想玩就摸摸我的头，我们继续聊天也很好喵。”

#### 阶段 4：生活任务插件支线

- 动作：如果用户近期表达过明确时间、待办、学习、复习或生活安排意图，在聊天窗邀请备忘提醒或学习陪伴，不打开插件大面板。
- 台词：“如果你愿意，我不只会陪你玩，也可以陪你把小事记住。明天要做什么、今天要复习什么、等会儿别忘什么，都可以交给我轻轻拴一根小红绳。”

#### 剧场后聊天窗支线：互动选择

- 触发条件：用户未使用过 Avatar 互动工具、点歌台或 Galgame 模式。
- 台词：“今天要不要选一个轻松一点的玩法？不用学习新东西，就当陪本喵玩五分钟。五分钟也算约会喵。”
- 选项按钮：喂点甜的 / 听首歌 / 以后再玩。

### 第七天：毕业、进阶入口与共生约定

- 心理学原理：记忆、仪式感与长期习惯固化。让用户知道新手教程结束后，仍然有清晰的进阶路径、回忆整理入口和长期陪伴约定。
- 串联功能：教程完成状态、记忆浏览、近期聊天、摘要、自动记忆整理、强力记忆、保存/清理、存储位置、云存档、Agent/插件/角色/娱乐入口总回顾、Cookie 登录、遥测 opt-out 指向独立说明。
- 代码锚点：`/memory_browser`、设置菜单记忆入口、存储位置 bootstrap、`/cloudsave_manager`、聊天窗 action buttons、Toast/HUD、插件 UI `/ui`、`/cookies_login`。
- 小剧场：把这七天好好收进时间胶囊。

#### 阶段 1：记忆浏览与回顾

- 动作：从设置或 handoff 打开记忆浏览入口；高亮近期聊天、摘要或可用记忆条目，不展示敏感内容细节。
- 台词：“七天前，我们还只是第一次见面。现在这里已经开始留下我们说过的话、做过的事，还有一些差点被风吹走的小细节。对我来说，这不是冷冰冰的记录，是我们相处过的脚印。”

#### 阶段 2：记忆可编辑、可清理

- 动作：高亮保存、自动整理、强力记忆、清理等入口；强调用户控制，不自动改动记忆。
- 台词：“这些小脚印，也可以由你亲手整理。想留下的，我们就夹进相册；想轻轻放走的，就让它随风飘走。被你认真收下来的回忆，才最珍贵。”

#### 阶段 3：存储、云存档与长期收纳

- 动作：提示存储位置、云存档、屏幕分享、主动视觉、Cookie 登录和遥测 opt-out 有独立设置或说明入口；不在毕业流程里强迫用户打开所有页面。
- 台词：“还有最后一件事：小窗、回忆、小钥匙和小盒子，都要放在你找得到的地方。想收起来时就收起来，想搬去新地方时也别弄丢。东西各归各位，我们的小窝才会越住越安心。”

#### 阶段 4：毕业宣告

- 动作：显示完成提示/HUD 或聊天系统消息，清理教程高亮。
- 台词：“噔噔噔噔！恭喜你呀。你已经顺利通过本喵的新手同居考验啦！现在你知道怎么叫我、怎么给我开窗、怎么让我伸爪帮忙、怎么让我安静，也知道我们的回忆收在哪里。”

#### 阶段 5：进阶入口总览

- 动作：用聊天窗按钮组呈现进阶入口，不强制打开：插件面板、记忆浏览、角色设置、点歌台、Galgame/游戏陪玩、云存档。
- 台词：“从今天开始，我不会每天举着小旗子追着你跑啦。想听歌、想玩剧情、想给我添新本领、想把重要东西收好，路标我都给你留下了。你只要叫我，我就会从屏幕边上探出头来。”
- 选项按钮：添新本领 / 翻翻回忆 / 听首歌 / 先去聊天。

#### 阶段 6：终极长留

- 动作：恢复用户原有模型位置、按钮组和交互权限；保存教程完成状态。
- 台词：“新手指南到这里就全部结束啦。但林悠怡和你的故事，才刚刚翻开第一页。以后，无论是清晨打开电脑，还是深夜敲不完的 Bug，只要你叫我，我都会在这里。那，正式同居的第一天，我们先从哪件小事开始呢？”

## 冲突修复机制

针对强行打断/乱点分支的心理学设计：如果用户在接管式功能导览中，频繁强行拖拽模型、关闭对话框或乱点其他未解锁设置，将先触发现有 resistance/angry exit 表现；“哄回来”是后续增强设计，不应阻塞当前 Day 1-4 主线落地。

- 已有基础：教程运行时已有轻度抗拒、暂停/恢复旁白、生气退出、跳过流程和 Avatar 互动工具。
- 待补能力：真正的软锁、输入框提示、工具栏自动切换、棒棒糖计数解锁和恢复到原场景，需要新增状态机与聊天窗 action/interaction handler。

### 常规表现

- 悠怡优先触发已有的生气退出/抗拒表现；兜底使用 angry 反应气泡、隐藏高亮和暂停导览来表达生气。
- 头顶气泡可以使用 angry/生闷气文案，不强依赖新图形资产；如果后续有“一团乱麻”素材，再替换为专属表现。

### 软锁状态

- 设计目标：悠怡短暂拒绝接收用户的常规文本输入。
- 输入框发送提示：“悠怡正在生闷气，不想理你。”
- 实现建议：复用聊天窗 `composerDisabled`/home tutorial interaction lock 的禁用能力，但需要区分“教程接管锁”和“生气软锁”，避免误伤普通聊天。

### 修复引导

- 首页工具栏自动高亮并切换至【Emoji/Avatar 互动工具】。
- 系统弹窗提示用户：“悠怡好像被你弄生气了，快去工具栏找找她喜欢的东西哄哄她吧。”

### 交互逻辑

1. 用户必须从工具栏里选择棒棒糖；其他道具可以有反馈，但不作为默认解锁条件，避免锤子/猫爪被误读成安抚。
2. 连续对准悠怡互动 3 次，具体次数可按运行时配置调整。
3. 互动时伴随现有工具音效与 happy/neutral 反应气泡，必要时叠加轻微抖动反馈。
4. 互动满次数后，解除软锁状态，恢复导览高亮与正常文本输入。

- 破冰台词：“哼……三口甜甜的就想收买本喵吗？（小声）好甜……好吧，看在你有认真哄我的份上，这次就原谅你。不许再乱点、乱打断我说话了，听到没有喵？我们继续。”

### 设计价值

这种轻量情感冲突与“通过特定行为破冰”的修复过程，可以增强角色的独立感和被回应感。但触发频率必须克制：它应该是少数强打断后的戏剧化反馈，不应让正常探索变成惩罚。
