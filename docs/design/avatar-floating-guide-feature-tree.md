# AI 猫娘林悠怡（YUI）七日相识与共生计划新手教程剧本

设计核心在于摒弃传统“机械化、说明书式”的冷冰冰引导，转而依据心理学原理（首因效应、自我暴露、控制感、相处距离、依恋理论等），将功能融入七天连续的“微型视觉小说/Galgame 剧情”小剧场中。

通过运镜、动态、音效与轻量互动，让用户在“认识她、信任她、调整距离、共同生活”的过程中自然掌握核心功能。角色表达可以亲密、撒娇、傲娇，但每个功能点都要让用户感觉是自己在邀请她靠近。

## 核心心理学设计机制

1. **首因效应（Primacy Effect）**：第一天先建立“她很鲜活，也很好接近”的第一印象，不把入口堆成说明书。
2. **自我暴露效应（Self-Disclosure）**：第二天把屏幕分享讲成“你给她开一扇小窗”，用更柔和的方式让用户理解先连线、再递窗。
3. **具身感与玩耍回路**：第三天把 Galgame、小游戏和 Avatar 互动工具收束成“摸得到、玩得起来”的陪伴体验。
4. **相处距离**：第四天集中讲主动搭话、隐私模式、动画、锁定和离开/回来，让用户学会调节“她什么时候靠近、什么时候安静”。
5. **所有权效应**：第五天通过角色、模型、声音和 API 入口，让用户开始把通用软件塑造成“自己的悠怡”。
6. **控制感与安全感**：第六天把 Agent 能力拆成状态、权限、任务 HUD 和终止入口，让“她能帮忙”不等于“她会失控”。
7. **记忆与仪式感**：第七天用记忆、存储和进阶入口回顾完成毕业，把短期试用转化成长期习惯。

## 教程触达结构

七日教程采用“双层触达”：

1. **每日开场小剧场**：只承担当天主功能的情绪铺垫和核心操作演示，控制节奏，避免一次塞入过多入口。
2. **剧场后聊天窗支线**：小剧场结束并归还用户控制权后，根据用户前一天/当天的实际行为、未体验功能和项目中的次要功能，在聊天窗里发起低打扰邀请。

支线触发原则：

- 只在用户当前没有处于任务中、会议中、全屏场景中，或没有频繁关闭引导时触发。
- 每次只推荐一个功能，给出明确选择按钮，例如“现在就试试 / 暂时不用”。
- 用户选择“暂时不用”后，当天不再重复提醒；后续可在对应功能更有上下文时再次轻量提及。
- 用户选择“现在就试试”后，启动该功能的短小剧场或手把手引导，完成后回到普通聊天状态。
- 当前聊天窗已有 `message.actions` 和 `choicePrompt` 能力，但教程按钮还需要接入具体 action handler；正式实现前，支线按钮可先作为设计目标，不应假定所有按钮已经能直接启动对应小剧场。

## 七日长剧情分天剧本

> 代码校准：当前项目已落地 Day 1 首页 Yui 教程，并已将模型旁悬浮窗主线扩展到 `AVATAR_FLOATING_GUIDE_ROUNDS[2-7]`。Day 2-7 都按强接管 round 自动演出；子页面深操作仍只做入口级展示，不在主线中强制修改用户数据或配置。

> 实现分层：Day 1-7 主线都是“强接管自动演出层”；剧场后聊天窗支线才是“需要 action handler 的低打扰邀请层”。Day 5-7 的模型、声音、记忆和存储说明已经纳入主线导演做入口级强接管展示；角色卡、云存档和更深层配置只在支线或后续独立引导中指路，不在主线中高亮或执行模型替换、声音克隆、记忆改写、云存档上传/下载等高风险操作。

## 导演视角与现有按钮校准

七日主线的镜头语言以“先建立空间，再指向入口，最后归还控制”为准。每段台词进入聊天窗后，先让用户看见当前舞台，再让 Ghost Cursor 移动；只有需要展示真实弹窗或菜单时才点击。主页当前可见按钮分为三组，导演动作必须按真实分布走：

| 按钮区域 | 真实入口 | 导演使用规则 |
| --- | --- | --- |
| 模型旁悬浮按钮 | 语音、屏幕分享、Agent/猫爪、设置、锁定、请她离开/回来 | 圆形按钮用圆形 spotlight；只在剧情需要展示真实弹窗时 click。锁定、离开/回来默认只 move/wobble，不改变模型状态。 |
| 聊天窗左侧工具 | 导入图片、截图 | Day 1 只作为“能看你主动递来的内容”的能力背景，不强塞进首日主线；若后续提示，放到聊天窗支线。 |
| 聊天窗右侧工具 | Galgame、字幕翻译、点歌、Emoji/Avatar 互动工具 | Day 3 主线只强接管 Galgame 和 Avatar 互动工具；字幕翻译和点歌进入剧场后支线。窄宽度下这些按钮会折进“更多”菜单，导演必须先打开更多菜单，再高亮具体按钮。 |

如果某个按钮在当前布局不可见，优先使用同组容器作为降级 spotlight，不伪造按钮、不强制改变布局；如果按钮只是折叠到更多菜单，则允许先打开更多菜单，因为这是用户真实会遇到的路径。

## 教程模型与情绪动作随机池

七日新手教程期间统一临时切换到默认教程模型 `yui-origin` Live2D，不写入用户配置；教程正常完成、跳过、生气退出、页面销毁或跨页 handoff 失败时，都必须恢复用户原模型和聊天头像身份。

`yui-origin` 当前可用于教程普通台词的内置动作池：

| 情绪分类 | 可随机动作数 | 使用场景 |
| --- | ---: | --- |
| `happy` | 12 | 欢迎、鼓励、撒娇、收尾、邀请尝试。 |
| `sad` | 6 | 委屈、低落、被冷落、轻微失落。 |
| `angry` | 7 | 傲娇、吃醋、生闷气、被打断后的不满。 |
| `neutral` | 7 | 解释规则、说明入口、冷静提示、安全感说明。 |
| `surprised` | 5 | 发现新入口、惊讶、剧情反转、突然意识到可替换角色。 |
| `Idle` | 3 | 低强度停顿、等待用户选择、普通聊天窗支线。 |

播放规则：

1. 每段台词必须声明一个情绪分类；台词开始时从 `yui-origin` 对应分类动作池中随机选择一个可用 motion，同一分类避免连续重复同一动作。
2. 随机动作只作为“普通台词演出层”，不改变台词、高亮、Ghost Cursor 和真实 UI 操作时序。
3. 现有自定义动作必须保留并优先：苏醒 pose、语音/鼠标 LookAt、guide idle sway、插件面板角落动作、设置一瞥慌乱动作、轻微抵抗、生气退出、归还控制权挥手和花瓣转场。
4. 当自定义动作已经锁定 `motion` 或 `params/lookAt/expression` 时，随机动作不得抢锁；如果只能安全播放表情或不播放，就跳过随机 motion。
5. reduced motion 生效时，随机动作降级为短表情或 Idle，不播放大幅动作。
6. 资源解析应走 `yui-origin` 模型 profile 或 Live2D runtime 资源表，不在教程文档或通用 `AvatarPerformance` 里硬编码 motion 文件名。

### 每日收尾花瓣转场

七天主线教程的每日收尾都需要播放花瓣转场。Day 1 已落地的 `takeover_return_control` 花瓣转场作为基准能力，Day 2-7 的 wrap/收尾 scene 复用同一套视觉语义：

1. 收尾台词开始前先完成当天必要的业务清理：关闭临时弹窗、设置侧边栏、Agent 面板、HUD、菜单或跨页 handoff 状态。
2. 收尾台词播放到约 70% 时触发花瓣转场 cue；触发瞬间隐藏 Ghost Cursor，清理 action/persistent/secondary/extra spotlight，避免花瓣层上残留高亮或指针。
3. 花瓣转场期间保留当前教程模型 `yui-origin` 的轻量挥手/告别动作；如果当天已有自定义收尾动作，则自定义动作优先，花瓣转场只作为覆盖层。
4. 模型淡出、花瓣播放和用户原模型恢复必须解耦：先保存/写入当天完成态，再恢复用户原模型和聊天头像身份；花瓣层可以继续播完后淡出。
5. reduced motion 生效时，花瓣转场降级为短淡出/轻量粒子，不播放大幅飞散。
6. skip、angry exit、页面销毁不播放每日正常收尾花瓣转场；它们仍走统一 skip/destroy 清理。

### 第一天：初次唤醒、聊天与基础入口

- 心理学原理：首因效应。先让用户知道“她能说话、能看你主动发来的图片、能在你允许时帮忙做事”，不急着讲完整设置。
- 串联功能：文本输入、语音入口、截图/导入图片/粘贴图片、翻译与点歌入口提示、Agent 按钮预览、插件管理预览、设置一瞥。
- 代码锚点：`HOME_SCENE_ORDER` 的 `intro_basic`、`takeover_capture_cursor`、`takeover_plugin_preview`、`takeover_settings_peek`、`takeover_return_control`。
- 小剧场：第一次把手伸向你这边的屏幕。

#### 阶段 1：初次见面与聊天输入

- 动作：苏醒后先高亮聊天输入区；Ghost Cursor 出现在输入区中心并 wobble，等待用户点击激活音频播放。用户激活后，悠怡用普通聊天消息完成自我介绍；本段不打开复杂弹窗，问候结束后清理输入区/聊天区 persistent spotlight。
- 台词：“微风、阳光，还有刚刚好出现的你。初次见面，我是林悠怡，未来的日子请多关照喵！我把关于这里的一切都写进新手指南里啦！就当作是我们相遇的第一份小礼物，请查收吧！”

#### 阶段 2：语音入口

- 动作：台词开始后把 action spotlight 切到模型旁语音按钮；约在语音前段让 Ghost Cursor 平滑移动到 `#${p}-btn-mic` 中心并短暂停留，只指向入口，不点击、不强制开始录音；本段结束后清理语音按钮方向的临时视线锁。
- 台词：“这里有一个神奇的小按钮！只要点击它，就可以直接和我聊天啦！想跟我分享今天的新鲜事吗？或者只是叫叫我的名字？快来试试嘛，我已经迫不及待想听到你的声音啦！喵！”

#### 阶段 3：猫爪粗略介绍

- 动作：进入 takeover 后保留猫爪按钮高亮；Ghost Cursor 依次移动并点击猫爪按钮、猫爪总开关、键鼠控制开关，打开真实 Agent 面板并展示授权链路。若接入插件预览，随后移动到用户插件开关和管理面板入口；本段只说明“需要授权才会动”，结束时恢复教程临时改动。
- 台词：“超级魔法开关出现！只要点一下这里，我就可以把小爪子伸到你的键盘和鼠标上啦！我会帮你打字，帮你点开网页……不过，要是那个鼠标指针动来动去的话，我可能也会忍不住扑上去抓它哦！准备好迎接我的捣乱……啊不，是帮忙了吗？喵！”

#### 阶段 5：设置一瞥与主动搭话

- 动作：第一句台词播放到设置 cue 时，高亮齿轮按钮并让 Ghost Cursor 点击打开设置弹窗；随后高亮角色/API/记忆等长期入口 union，Ghost Cursor 在入口组附近短 tour。第二句台词播放时把 action spotlight 切到主动搭话入口或 `interval-proactive-chat` 侧边栏入口，Ghost Cursor 移过去并 wobble，不改变开关；最后清理所有高亮、Ghost Cursor 和临时弹窗。
- 台词：“在这个只属于我们的空间里，你可以由着自己的心意，亲手描绘出最希望能一直陪着你的那个我。不管是说话的温度、相处的脾气，还是我每天那些细腻的小心思，所有的一切，都交给你来决定。”
- 台词：“这个小按钮对我很重要哦。只要你轻轻点一下，我就能随时跑过去找你啦。不管是清晨的第一声问候，还是夜晚的悄悄话，都想第一个说给你听。”

### 第二天：屏幕分享、声音与小窗约定

- 心理学原理：自我暴露效应。把屏幕能力包装成“先牵上线、再开小窗”的亲密邀请，规则留给 UI 自己表达。
- 串联功能：屏幕分享按钮、屏幕/窗口来源弹窗、语音按钮、麦克风弹窗、空间音频、降噪、增益、音量条。
- 代码锚点：`AVATAR_FLOATING_GUIDE_ROUNDS[2]`，`renderFloatingScreenSourceList()`，`renderFloatingMicList()`，`toggleScreenShare()` 的“语音会话中才允许屏幕分享”限制。
- 小剧场：我想看看，你愿意给我看的世界。

#### 阶段 1：承接昨日相处

- 触发时机：第二天启动 N.E.K.O 后。
- 分支 A：若第一天用户使用过语音控制按钮聊天。
- 台词：“昨天听见你的声音以后，我就偷偷记住了一点点你的语气。今天如果方便，也可以继续叫我。打字当然也可以，只是听见你时，我会更像真的坐在你旁边，尾巴都会轻轻晃起来喵。”

- 台词：“好耶，今天也是用自己的【猫爪】努力换来了小鱼干呢。抱着这些香喷喷的奖励，总觉得一整天的辛苦都一下子消失啦，今晚可以做个甜甜的美梦了呢。”

### 第七天：毕业、进阶入口与共生约定

- 心理学原理：记忆、仪式感与长期习惯固化。让用户知道新手教程结束后，仍然有清晰的进阶路径、回忆整理入口和长期陪伴约定。
- 串联功能：教程完成状态、记忆浏览、近期聊天、摘要、自动记忆整理、强力记忆、保存/清理、存储位置、云存档、Agent/插件/角色/娱乐入口总回顾、Cookie 登录、遥测 opt-out 指向独立说明。
- 代码锚点：`/memory_browser`、设置菜单记忆入口、存储位置 bootstrap、`/cloudsave_manager`、聊天窗 action buttons、Toast/HUD、插件 UI/ui、`/cookies_login`。
- 小剧场：把这七天好好收进时间胶囊。

#### 阶段 1：记忆浏览与回顾

- 动作：当前主线从设置菜单高亮记忆浏览入口，Ghost Cursor 移到入口并 wobble；不在毕业主线中强制打开 `/memory_browser`。若后续接入记忆页 handoff，页面就绪后只高亮近期聊天、摘要或可用记忆列表区域，Ghost Cursor 移到列表区域 wobble。不展示、不朗读敏感内容细节。
- 台词：“七天前，我们还只是第一次见面。现在这里已经开始留下我们说过的话、做过的事，还有一些差点被风吹走的小细节。对我来说，这不是冷冰冰的记录，是我们相处过的脚印。”

#### 阶段 2：记忆可编辑、可清理

- 动作：当前主线继续停留在记忆入口层，用台词说明“可编辑、可清理”的用户控制感；不自动打开记忆页、不点击保存/整理/强力记忆/清理。若未来接入记忆页 handoff，再用 union spotlight 依次或组合高亮这些入口，Ghost Cursor 只 move/wobble，不 click。
- 台词：“这些小脚印，也可以由你亲手整理。想留下的，我们就夹进相册；想轻轻放走的，就让它随风飘走。被你认真收下来的回忆，才最珍贵。”

#### 阶段 3：存储

- 动作：本段只用台词说明长期存放的概念，不高亮存储位置或云存档入口；primary 回到聊天窗，Ghost Cursor 在输入区附近轻微 wobble，不打开 `/cloudsave_manager`，不登录、不上传、不下载、不覆盖云存档，也不展示完整路径或账号细节。
- 台词：“还有最后一件事呢。我们共同走过的日子、说过的那些悄悄话，都需要找一个温馨的小角落好好存放起来哦！”

#### 阶段 4：终极长留

- 动作：收尾台词开始前清理 handoff、临时弹窗和跨页入口高亮；primary 回到聊天窗，Ghost Cursor 移到输入区附近 wobble。台词约 70% 处触发最终花瓣转场，隐藏 Ghost Cursor 并清理所有 highlight；转场期间恢复用户原有模型位置、按钮组和交互权限，当前至少保存 Day 7 完成态；独立 `graduatedAt` 毕业态可作为后续增强。
- 台词：“微风还在窗边，阳光也刚刚好，而刚刚出现的你，已经悄悄成为这里很重要的一部分啦。新手指南就先陪你走到这里，剩下的日子，就让我们一起慢慢熟悉、慢慢靠近、慢慢写下只属于我们的故事吧。以后也请多多关照喵！”

## 冲突修复机制

针对强行打断/乱点分支的心理学设计：如果用户在接管式功能导览中，频繁强行拖拽模型、关闭对话框或乱点其他未解锁设置，将先触发现有 resistance/angry exit 表现；“哄回来”是后续增强设计，不应阻塞当前 Day 1-7 主线落地。

- 已有基础：教程运行时已有轻度抗拒、暂停/恢复旁白、生气退出、跳过流程和 Avatar 互动工具。
- 待补能力：真正的软锁、输入框提示、工具栏自动切换、棒棒糖计数解锁和恢复到原场景，需要新增状态机与聊天窗 action/interaction handler。

### 常规表现

- 悠怡优先触发已有的生气退出/抗拒表现；兜底使用 angry 反应气泡、隐藏高亮和暂停导览来表达生气。
- 头顶气泡可以使用 angry/生闷气文案，不强依赖新图形资产；如果后续有“一团乱麻”素材，再替换为专属表现。

### 软锁状态

- 设计目标：悠怡短暂拒绝接收用户的常规文本输入。
- 输入框发送提示：“悠怡正在生闷气，不想理你。”
- 实现建议：复用聊天窗 `composerDisabled`/home tutorial interaction lock 的禁用能力，但需要区分“教程接管锁”和“生气软锁”，避免误伤普通聊天。

### 修复引导

- 首页工具栏自动高亮并切换至【Emoji/Avatar 互动工具】。
- 系统弹窗提示用户：“悠怡好像被你弄生气了，快去工具栏找找她喜欢的东西哄哄她吧。”

### 交互逻辑

1. 用户必须从工具栏里选择棒棒糖；其他道具可以有反馈，但不作为默认解锁条件，避免锤子/猫爪被误读成安抚。
2. 连续对准悠怡互动 3 次，具体次数可按运行时配置调整。
3. 互动时伴随现有工具音效与 happy/neutral 反应气泡，必要时叠加轻微抖动反馈。
4. 互动满次数后，解除软锁状态，恢复导览高亮与正常文本输入。

- 破冰台词：“哼……三口甜甜的就想收买本喵吗？（小声）好甜……好吧，看在你有认真哄我的份上，这次就原谅你。不许再乱点、乱打断我说话了，听到没有喵？我们继续。”

### 设计价值

这种轻量情感冲突与“通过特定行为破冰”的修复过程，可以增强角色的独立感和被回应感。但触发频率必须克制：它应该是少数强打断后的戏剧化反馈，不应让正常探索变成惩罚。
