# 米家智能家居插件 — 快速指引

> **教程定位**：本教程面向希望在 N.E.K.O 平台上通过自然语言控制米家智能设备的用户。完成本教程后，您将能够用一句话开关灯、调空调、执行场景，无需记忆任何设备 ID 或技术参数。

---

## 最终成果

完成本教程后，您将拥有：

1. 一个已登录米家账号的 N.E.K.O 插件实例
2. 能通过自然语言（如"打开卧室灯"）控制家中米家设备
3. 能查询设备状态、执行米家场景
4. 了解消歧机制，能精确控制同名设备

---

## 目标读者

- 拥有米家（小米）智能设备,拥有米家账号的用户
- 已完成 N.E.K.O 主程序安装并能正常运行

## 环境搭建（github源码版本专属，其他版本直接运作exe文件启动即可）

### 第 1 步：确认 N.E.K.O 已就绪
```
环境要求：Python 3.11（不支持其他版本）、uv 包管理器、Node.js（>=20.19）

# 1. 克隆项目
git clone https://github.com/Project-N-E-K-O/N.E.K.O.git
cd N.E.K.O

# 2. 安装 Python 依赖
uv sync

# 3. 构建前端项目（需要 Node.js >= 20.19，首次运行或前端代码变更后需要）
#    推荐：使用一键脚本（这是官方支持的构建方式）
#      Windows：      build_frontend.bat
#      Linux/macOS：  ./build_frontend.sh
#    如需手动构建（命令须与脚本保持一致）：
# cd frontend/react-neko-chat && npm install && npm run build && cd ../..
# cd frontend/plugin-manager && npm install && npm run build-only && cd ../..

# 4. 启动服务（至少需要 main_server 和 memory_server）
uv run python memory_server.py
uv run python main_server.py
# 可选：启动 Agent 服务
uv run python agent_server.py

# 5. 访问 http://localhost:48911 配置 API Key 并开始使用
开发者建议加入企鹅群 995414391 交流。

### 1. 启动 N.E.K.O 主程序

```bash
cd /path/to/N.E.K.O
python main.py
```

### 2. 打开插件面板

在 N.E.K.O 的 用户插件 中找到「米家智能家居」面板。

在上方导航栏点击面板即可看到插件界面。

### 3. 扫码登录米家账号

在插件面板中找到「登录」区域，点击「扫码登录」按钮。面板会显示一个二维码。

打开手机上的**米家 App** → 点击右上角 `+` → 扫一扫 → 扫描屏幕上的二维码 → 在手机上确认登录。

### 4. 验证登录成功

登录成功后，面板会显示您的米家用户名和已绑定的家庭列表。此时可以关闭登录面板。

### 5. 用自然语言控制设备

在 N.E.K.O 的 AI 对话中输入：

```
打开卧室灯
```

如果您的米家账号下有名为"卧室灯"或位于"卧室"的灯设备，它将被打开。

**验证成功的标志**：终端日志显示设备控制成功的返回信息，设备实际状态发生变化（灯亮起）。

## 迭代式讲解：从基础到进阶

### 第一阶段：基础开关控制

**目的**：让设备响应最简单的"打开/关闭"指令。（本插件对于自然语言处理能力弱，对于开下开关口语表达可能误判）

在 AI 对话中尝试：

```
打开客厅灯
关闭卧室灯
```

**改动说明**：无需修改任何代码，直接使用自然语言即可。

**预期结果**：对应设备执行开关操作。如果家中只有一个灯，直接说"打开灯"也可以。

---

### 第二阶段：属性调节

**目的**：控制设备的具体参数，如亮度、温度。

尝试以下指令：

```
灯亮度调到50%
空调调到26度
风扇调到3档
```

**预期结果**：设备属性被设置为指定值。插件会自动解析数字和单位。

---

### 第三阶段：模式切换与场景执行

**目的**：切换设备工作模式或执行预设场景。

```
空调调制冷模式
执行回家模式
```

**注意**：场景需要先在米家 App 中创建好。插件会列出所有可用场景，匹配名称后执行。

---

### 第四阶段：处理多设备消歧

**目的**：当家中有多个同名设备时，精确指定目标。

如果您家中有多个"灯"，直接说"打开灯"会触发消歧提示：

```
找到 3 个匹配 '灯' 的设备：
1. 🟢 客厅 吸顶灯
2. 🔴 卧室 吸顶灯
请用房间名+设备名精确指定，如 '卧室灯'
```

此时补充房间名即可：

```
打开卧室吸顶灯
```

**推荐习惯**：始终使用 `房间名 + 设备名` 的格式，如"客厅灯"、"主卧空调"。

---

### 第五阶段：查询设备状态

**目的**：了解设备当前状态，而非仅控制。

```
空调现在多少度？
卧室灯开着吗？
```

插件会返回设备的当前属性值（温度、亮度、开关状态等）。

## API 入口参考（供高级用户）

如果您需要直接调用插件接口（而非通过自然语言），以下是核心入口：

| 入口 ID | 功能 | 参数示例 |
|---------|------|---------|
| `smart_control` | 自然语言统一控制（推荐） | `{ "command": "打开卧室灯" }` |
| `find_device_by_name` | 按名称查找设备 | `{ "name": "卧室灯" }` |
| `query_device_state` | 查询设备详细状态 | `{ "name": "空调" }` |
| `execute_scene` | 执行智能场景 | `{ "scene_id": "12345" }` |

## 故障排查

### 登录相关

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 扫码后无反应 | 二维码已过期 | 重新点击「扫码登录」生成新二维码（2 分钟内有效） |
| 提示"未登录" | 凭据文件损坏或被删除 | 重新扫码登录，凭据会自动保存到 `data/credential.json` |
| 登录后设备列表为空 | 账号下未绑定设备 | 在米家 App 中确认设备已添加到当前账号 |

### 设备控制相关

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 提示"未找到设备" | 设备名称不匹配 | 先用"查询"功能确认设备名称，或在米家 App 中查看 |
| 指令无响应 | 设备离线 | 检查设备是否通电并连接网络，在米家 App 中确认在线状态 |
| 消歧提示太多 | 家中同名设备过多 | 使用 `房间名 + 设备名` 格式精确指定 |
| 控制超时 | 网络不稳定 | 检查网络连接，稍后重试 |

### 插件启动相关

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 插件未出现在面板中 | 插件未启用 | 在 N.E.K.O 设置中确认 `mijia` 插件已启用 |
| 启动时报 ImportError | Python 版本过低 | 升级到 Python 3.11+ |
| 启动时报依赖缺失 | N.E.K.O 环境不完整 | 重新安装 N.E.K.O 依赖：`pip install -r requirements.txt` |

## 完整配置参考

插件配置文件位于 `plugin/plugins/mijia/plugin.toml`，以下为关键字段：

```toml
[plugin]
id = "mijia"
name = "米家智能家居"
entry = "plugin.plugins.mijia:MijiaPlugin"

[plugin_runtime]
enabled = true
auto_start = false
```

- `enabled = true`：插件已启用
- `auto_start = false`：插件不会自动启动，需在面板中手动开启或通过 AI 指令触发

凭据文件位于 `plugin/plugins/mijia/data/credential.json`，由插件自动管理，无需手动编辑。

完成基本功能后，可以尝试以下进阶任务：
