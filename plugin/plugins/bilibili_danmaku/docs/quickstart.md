# B站弹幕监听插件 — 快速指引

> **教程定位**：本教程面向希望在 N.E.K.O 平台上实时监听 B站直播间弹幕、并通过 AI 自动回复的用户。完成本教程后，您将能够连接任意 B站直播间，实时接收弹幕，让 AI 根据弹幕内容生成回复。

---

## 最终成果

完成本教程后，您将拥有：

1. 一个已连接 B站直播间的弹幕监听实例
2. 实时弹幕流显示，包含弹幕、礼物、SC（Super Chat）
3. AI 自动根据弹幕内容生成并发送回复（可选）
4. 背景 LLM 引导词生成能力（可选，进阶功能）
5. B站读写工具（搜索视频、发评论等）

---

## 目标读者

- 有 B站账号的用户（游客模式也可监听，但无法发送弹幕）
- 已完成 N.E.K.O 主程序安装并能正常运行
- 不需要编程基础，能操作浏览器和手机即可
- 了解 B站直播间基本概念（房间号、弹幕、礼物、SC）

## 迭代式讲解：从基础到进阶

### 第一阶段：游客模式监听（无需登录）

**目的**：在不登录 B站账号的情况下监听弹幕。

按照上述 MVP 步骤操作即可。游客模式下：

- 可以接收弹幕、礼物、SC
- 无法发送弹幕
- 无法使用高级过滤（如用户等级过滤）

**预期结果**：弹幕流实时显示直播间内容。

---

### 第二阶段：登录 B站账号（宅久大佬提供技术支持）

**目的**：解锁发送弹幕、高级过滤等功能。

在插件面板的「B站账号」区域：

1. 点击「扫码登录」按钮，面板会显示一个二维码
2. 打开手机上的 **B站 App** → 左上角扫一扫 → 扫描屏幕上的二维码 → 在手机上确认登录
3. 登录成功后面板会显示您的 B站用户名和 UID

**注意**：二维码 2 分钟内有效，过期后需要重新点击「扫码登录」。

**预期结果**：面板显示已登录状态，用户名和 UID 正确。

---

### 第三阶段：AI 自动回复弹幕（由宅久大佬完成）

**目的**：让 AI 根据弹幕内容自动生成回复并发送到直播间。

在插件面板的「AI 推送设置」区域：

1. 设置 **推送间隔**：建议 `15~30` 秒（太短 AI 响应频繁，太长回复延迟大）
2. 设置 **弹幕最大长度**：保持默认 `20`（B站限制 20 字符）
3. 设置 **目标 AI 名称**（可选）：指定哪个 AI 接收弹幕推送，留空则推送给默认 AI
4. 设置 **主人 B站 UID**（可选）：设置后 AI 会对主人消息特殊处理

**预期结果**：直播间观众发送弹幕后，AI 会在推送间隔内生成回复并自动发送到直播间。

---

### 第四阶段：背景 LLM 引导词（进阶）

**目的**：让弹幕先经过一个背景 LLM 分析，生成更贴合直播氛围的引导词，再交给主 AI 处理。

在插件面板的「背景 LLM 设置」区域：

1. 打开「启用开关」
2. 填写 **API 地址**：如 `https://api.deepseek.com/v1/chat/completions`
3. 填写 **模型名称**：如 `deepseek-v4-flash`
4. 填写 **API Key**：您的 API 密钥
5. 设置 **聚合窗口**：建议 `10~20`（收集多少条弹幕后触发一次 LLM 总结）
6. 点击「保存配置」
7. 点击「🔍 测试」验证 API 连通性

**进阶设置**（点击「进阶设置」展开）：

| 设置项 | 说明 |
|--------|------|
| 猫娘名字 | 用于提示词中的 `{name}` 占位符替换 |
| 知识库上下文 | 角色人设、口头禅、常见梗 |
| 提示词模板 | 自定义 System Prompt，留空使用默认模板 |

**预期结果**：弹幕不再直接推送给主 AI，而是先由背景 LLM 分析直播氛围、生成引导词，再交给主 AI 生成更自然的回复。

---

### 第五阶段：使用 B站读写工具

**目的**：通过api直接读取 B站数据或执行写操作。

#### 读工具（无需登录）

在面板的「B站读工具」区域，填写关键词或 BV 号，点击对应按钮：

| 工具 | 用途 | 必填参数 |
|------|------|---------|
| 搜索视频 | 按关键词搜索 | 关键词 |
| 热门视频 | 全站热门列表 | 无 |
| 视频信息 | 获取视频详情 | BV 号 |
| 视频评论 | 获取评论列表 | BV 号 |
| 用户信息 | 获取用户资料 | UID |

#### 写工具（需要登录，谨慎使用）

| 工具 | 用途 | 必填参数 |
|------|------|---------|
| 发评论 | 在视频下发评论 | BV 号 + 评论内容 |
| 发动态 | 发布新动态 | 动态文本 |
| 发私信 | 向用户发私信 | 对方 UID + 内容 |

**注意**：写工具会对您的 B站账号产生实际影响，请确认内容无误后再操作。

## 常见疑问

**Q：游客模式和登录模式有什么区别？**

A：游客模式可以监听弹幕，但无法发送弹幕、无法使用用户等级过滤、无法使用写工具。登录后解锁全部功能。

**Q：推送间隔设多少合适？**

A：建议 15~30 秒。间隔太短会导致 AI 频繁响应、消耗 token 太多；间隔太长会导致回复延迟明显。可以根据直播间弹幕密度调整。

**Q：背景 LLM 和主 AI 是什么关系？**

A：背景 LLM 负责分析弹幕氛围、生成引导词（如"观众在讨论游戏"），主 AI 根据引导词生成最终回复。背景 LLM 是可选的增强功能，不启用时弹幕直接推送给主 AI。

**Q：一个插件实例能监听多个直播间吗？**

A：不能。每个插件实例连接一个直播间。

**Q：弹幕监听断线了怎么办？**

A：插件内置自动重连机制（最多 10 次，指数退避）。如果自动重连失败，可以手动点击「▶ 开始监听」重新连接。

## 故障排查

### 连接相关

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 状态灯为灰色 | 未开始监听 | 点击「▶ 开始监听」按钮 |
| 状态灯为红色 | 连接失败 | 检查网络是否能访问 B站，检查房间号是否正确 |
| 状态灯为黄色 | 正在连接中 | 等待几秒，如果持续为黄色则检查网络 |
| 连接后无弹幕 | 直播间无观众或已下播 | 确认直播间正在直播且有观众互动 |
| 频繁断线重连 | 网络不稳定 | 检查网络连接质量，尝试有线网络 |

### 登录相关

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 扫码后无反应 | 二维码已过期 | 重新点击「扫码登录」生成新二维码 |
| 提示"凭据无效" | Cookie 过期 | 重新扫码登录 |
| 登录后无法发弹幕 | 直播间限制 | 部分直播间限制账号等级或粉丝牌，检查直播间规则 |

### AI 回复相关

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| AI 不回复弹幕 | 推送间隔未设置 | 确认「推送间隔」已设置且不为 0 |
| AI 回复延迟大 | 推送间隔太长 | 缩短推送间隔（建议不低于 10 秒） |
| 背景 LLM 不工作 | API 配置错误 | 点击「🔍 测试」检查 API 连通性 |
| 发送弹幕失败 | 未登录或无权限 | 确认已登录且有发弹幕权限 |

## 完整配置参考

### 基础配置

文件路径：`plugin/plugins/bilibili_danmaku/data/config.json`

```json
{
  "room_id": 22925943,
  "interval_seconds": 30,
  "target_lanlan": "",
  "danmaku_max_length": 20,
  "master_bili_uid": 0,
  "master_bili_name": ""
}
```

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `room_id` | 直播间 ID，`0` 表示不自动连接 | `0` |
| `interval_seconds` | AI 推送间隔（秒），范围 5~180 | `30` |
| `target_lanlan` | 目标 AI 名称，留空推送给默认 AI | `""` |
| `danmaku_max_length` | 弹幕最大字符数 | `20` |
| `master_bili_uid` | 主人 B站 UID，`0` 表示未设置 | `0` |
| `master_bili_name` | 主人 B站用户名 | `""` |

### 背景 LLM 配置

文件路径：`plugin/plugins/bilibili_danmaku/data/config_enhanced.json`

```json
{
  "background_llm": {
    "enabled": true,
    "cloud": {
      "url": "https://api.deepseek.com/v1/chat/completions",
      "api_key": "sk-your-key-here",
      "model": "deepseek-v4-flash",
      "timeout_sec": 10,
      "retry_times": 2
    },
    "window_size": 15,
    "max_samples": 30,
    "knowledge_context": "",
    "neko_name": "",
    "prompt_template": ""
  }
}
```

| 字段 | 说明 |
|------|------|
| `enabled` | 是否启用背景 LLM |
| `cloud.url` | 兼容 OpenAI 格式的 API 端点 |
| `cloud.api_key` | API 密钥 |
| `cloud.model` | 模型名称 |
| `cloud.timeout_sec` | 请求超时时间（秒） |
| `cloud.retry_times` | 失败重试次数 |
| `window_size` | 聚合多少条弹幕触发一次 LLM |
| `max_samples` | 弹幕采样池最大容量 |
| `knowledge_context` | 角色人设、口头禅等上下文 |
| `neko_name` | 猫娘名字，用于 `{name}` 占位符 |
| `prompt_template` | 自定义 System Prompt，留空使用默认 |
