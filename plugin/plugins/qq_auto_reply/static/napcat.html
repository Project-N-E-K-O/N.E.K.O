配置引导
1. 下载NapCat：点击下方链接，或自行搜索NapCat仓库，下载对应的版本（推荐使用NapCat.Shell）
NapCat 下载
2. 打开 NapCat 终端上 [WebUI] 标签的网址：
NapCat 终端
3. 打开的页面如下，点击「网络配置」→「新建」→「WebSocket客户端」
4. 在「连接地址」中填写 N.E.K.O 的反向 WS 地址（如 ws://127.0.0.1:6199/ws）和 Token（注意：如果后面自动回复没触发，回来检查 Token 是否与配置文件一致），输入名称 → 勾选启用 → 保存
1. 配置反向 WS 监听地址：上方「通信地址 (URL)」默认 ws://0.0.0.0:6199。N.E.K.O 将在此地址启动 WebSocket 服务器。在 NapCat 的 WebUI 中创建「WebSocket 客户端」连接到此地址（如 ws://127.0.0.1:6199/ws）
2. 填入访问令牌：将 NapCat 配置的 Token 输入到「访问令牌 (TOKEN)」字段。N.E.K.O 会验证连接客户端的 token，与 NapCat 中不一致则拒绝连接。
3. 配置 NapCat 目录：插件可以通过绝对路径运行 NapCat，在「NapCat 目录」输入路径或点击「浏览...」选择目录即可
4. 点击侧边栏的保存设置。点「启动」后 N.E.K.O 会开启 WebSocket 服务端，NapCat 需要至多 30 秒才会发起连接（它有重试间隔），请耐心等待状态页的 OneBot 指示灯变绿。
→ 快速跳转到连接配置
1. 添加管理员：在「配置 → 账号」页中点击「+ 添加」按钮，级别选择管理员，输入你的 QQ 号，确认添加。管理员是唯一能触发自动回复的私聊用户。
2. 添加其他用户：需要回复的其他用户，级别选择「信任用户」，输入对应的 QQ 号即可。
→ 快速跳转到账号页
1. 启动自动回复：点击侧边栏底部的「启动」按钮，插件会自动连接 OneBot 并开始监听消息。启动成功后状态页会显示「已连接」。
2. 查看运行状态：切换到「日志」页查看实时运行日志，确认是否收到消息、是否成功生成回复。如果出现异常，日志会显示具体错误信息。
→ 快速跳转到日志页
1. 上传表情包：切换到「表情包」页，拖入图片或点击选择文件，填写描述后点上传。支持拖入整个文件夹批量导入。
2. 使用表情包：LLM 会在 System Prompt 中看到已注册的表情包列表，在对话中自动选择合适的表情包用 <sticker>ID</sticker> 标签发送。
3. 管理：已注册的表情包显示在下方列表，数据存储在 data/sticker.json，图片在 data/sticker/。
→ 快速跳转到表情包页
1. 查看未读消息：切换到「审阅」页查看各群聊的未读消息概要。消息按标签分类（点名、问题、反馈），可通过关键词标签配置分类规则。
2. 处理消息：点击群名展开消息详情，可逐条审阅。审完后点击「标记已处理」清除未读计数。猫娘动态模式下焦点群切换时自动回溯补回忽略的消息。
3. 关键词标签：在 配置 → 关键词 中设置标签。正优先级触发注意力爆发，负优先级 = 黑名单，命中后不记录不回复。
→ 快速跳转到审阅页
1. 策略模式：在 配置 → 策略 中选择模式。「猫娘动态」通过多群注意力竞争驱动回复，「猫娘场景」使用 @bot + 概率门控。
2. 注意力机制：每个群有独立注意力条（0~10）。消息增益、关键词爆发、回复消耗、时间衰减。焦点群获得优先回复权。
3. 回溯补回：焦点切换到新群时自动回溯被忽略的消息，通过摘要让 LLM 判断是否需要补回复。
→ 快速跳转到策略页
Linux / Docker 部署
宿主机 NapCat + Docker N.E.K.O:
1. 宿主机安装启动 NapCat
2. NapCat WebUI→网络配置→新建→WS客户端
3. URL: ws://宿主机IP:6199/ws
4. compose 加 - "127.0.0.1:6199:6199"（仅本机）或 - "6199:6199"（内网，需配 Token）
5. Token 一致
Docker NapCat + Docker N.E.K.O:
1. 同网络 → NapCat WS客户端 URL: ws://容器名:6199/ws（容器名即 compose 中 service 名称）
Kubernetes Sidecar:
1. 同 Pod → NapCat WS客户端 URL: ws://localhost:6199/ws
⚠️ N.E.K.O 容器内不支持运行 NapCat（Windows cmd.exe），需单独部署。
1. 自动回复没反应：检查 OneBot 是否已连接（状态页）、Token 是否与 NapCat 中一致、信任用户是否已添加。
2. 发送消息失败：检查 QQ 是否已登录（NapCat 扫码）、群聊是否在信任列表中。
3. 查看日志：切换到「日志」页查看实时运行日志，错误信息会详细记录。开启实时模式可自动刷新。
4. 配置文件位置：所有配置存储在 business_config.json（可通过 N.E.K.O 数据目录找到），出问题时可直接查看。
