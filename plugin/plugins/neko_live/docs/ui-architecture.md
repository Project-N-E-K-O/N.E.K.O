# NEKO Live UI 架构

本文是 Hosted UI 的长期约束。具体按钮与主播操作以 [快速开始](quickstart.md) 为准。

## 页面分区

普通模式固定为四个一级页面：

1. **控制台**：账号与平台、直播间二次确认、主题、节奏、关键状态、开始/结束直播。
2. **直播间互动**：首次出场、后续弹幕、支持事件、主持能力及其详细设置。
3. **观众**：本场直播数据与观众档案，不承载技术诊断。
4. **设置**：安全与性能、数据与隐私、帮助与高级。

开发者模式开启后条件追加 **开发者工具**。未完成的私信、自动化或平台写能力不占一级导航。

控制台应让主播完成整场直播的必需操作；详细参数、资料和诊断进入对应页面或弹窗。不要把所有功能重新塞回一个长页面。

## 源码与兼容入口

- `ui/panel.tsx`：模块化面板入口。
- `ui/panel_components.tsx`：通用展示组件和模块错误边界。
- `ui/panel_data_sections.tsx`：观众与诊断数据区段。
- `ui/panel_state.ts`：状态类型、表单默认值和草稿契约。
- `ui/panel_helpers.ts`：标签与格式化 helper。
- `ui/panel_compat.tsx`：manifest 实际加载的完整单文件兼容入口。

`panel_compat.tsx` 只能内联模块化源码的完整行为，不能成为另一套实现，也不能替换成只剩状态概览的最小壳。修改源码后必须重新生成兼容入口，并用测试和 diff 确认行为一致。

## 模块贡献

模块可以通过 `domain` 和 `config_schema()` 向互动页贡献设置。当前支持的字段类型只应覆盖已有真实消费者；新增类型时要同时补后端清洗、Hosted UI 渲染、8 locale 和测试。

功能参数跟功能卡走，平台级安全与隐私设置留在设置页，开播频繁操作留在控制台。原始队列数字、内部阈值和 dry-run 不属于普通主播日常设置。

每张模块卡必须有独立错误边界。一个模块的 schema、状态或渲染失败时只降级该卡，不能让整个面板黑屏。

## 表单与弹窗

- 弹窗打开时从已保存快照初始化本地草稿；
- 输入过程只修改草稿，不能每输入一个字符重建整个表单或跳焦点；
- 只有“保存设置”写入，关闭、取消或遮罩退出都丢弃草稿；
- 保存成功后更新快照，再关闭弹窗；失败时保留草稿并显示原因；
- 不依赖可变标题、当前输入值或数组索引作为组件 key；
- 关闭功能时保持卡片基本尺寸稳定，详细设置可禁用但不让布局大幅跳动。

直播间确认是两步流程：先查询并展示房间号、标题、主播和直播状态，再由用户确认。输入变化会使旧查询结果失效。

## 开始/结束直播控制

主控制应固定在插件面板右下可见区域，不随内部内容滚动离开视口。它必须：

- 明显区分未准备、可开始、正在开始、直播中、正在结束和失败；
- 未准备时不可点击，并按“登录账号 → 确认直播间 → 检查输出/暂停”顺序提示下一步；
- 开始前二次确认平台、账号、房间和模式；
- 直播中明确显示状态，点击执行结束流程而不是再次开始；
- pending 期间锁定重复操作。

## 状态刷新与性能

- 只在 `document.visibilityState === "visible"` 且直播相关状态需要刷新时轮询；
- 同一时刻只允许一个 refresh 请求；
- 使用可清理的递归 timeout，不新增常驻高频 interval；
- 隐藏或卸载时清理 timer 和监听；重新可见时立即补刷一次；
- action 成功与随后 refresh 失败分开处理，不能把成功动作改报失败；
- 连续刷新失败显示“状态可能过期”和最后成功刷新时间。

静态内容、档案列表和非当前子页不应持续刷新。面板渲染成本必须低于直播输入、输出和 TTS 主链路。

## Hosted UI 限制

- 使用 `@neko/plugin-ui` 暴露的组件与 hooks；不要假设存在浏览器框架的完整 API。
- 当前运行时不支持 `useRef` 和 SVG 命名空间渲染。
- `<img src="data:...">` 会被 URL 安全规则过滤；不要把头像 bytes/base64 放进 UI。
- 内部胶囊式子页应实现 tab 键盘语义，并支持方向键、Home、End。
- 日期和时间使用插件当前 locale，不使用浏览器默认语言。
- 新增用户可见文案同步 8 个 locale 文件。

## 安全边界

UI 只读取 dashboard 投影并调用声明权限内的 action/config API。它不得：

- 直接读取凭据或 store 文件；
- 绕过 runtime、pipeline、SafetyGuard 或 Dispatcher；
- 展示 cookie、token、raw payload、原始弹幕、头像 bytes/base64；
- 用前端禁用代替后端权限检查。

## 验证

UI 改动至少检查：

- 模块化源码与 `panel_compat.tsx` 行为一致；
- Hosted TSX 可按生产选项转译；
- 四个普通一级页和条件开发者页顺序正确；
- 弹窗草稿、输入焦点、保存失败和快照恢复；
- 重复点击、连接中换房、结束直播和 refresh 失败；
- 页面隐藏后的 timer 回收与单请求刷新；
- 8 locale key 与占位符一致；
- 插件完整测试与 CLI check。
