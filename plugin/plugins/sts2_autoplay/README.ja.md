# クイックスタート

`sts2_autoplay` は、`STS2 AI Agent` が公開するローカルの『Slay the Spire 2』状態を N.E.K.O に接続するために使用します。プラグインは盤面を読み取り、合法な行動を実行し、戦略に従って自動プレイし、ねこ娘に 1 枚のカードを選ばせ、フロントエンドに観察情報をプッシュし、バックグラウンドタスクの中でねこ娘がソフトガイダンスを送信して次の意思決定ラウンドに影響を与えることを許可します。

## 使い方チュートリアル

### MOD の取得

Git Clone を使用：
```text
git clone https://github.com/CharTyr/STS2-Agent.git
```

### ゲーム Mod のインストール

Steam で *Slay the Spire 2* を右クリックし、「管理 -> ローカルファイルを閲覧」を選択できます。

Steam のデフォルトのゲームディレクトリは通常以下のような感じです：

```text
...\Steam\steamapps\common\Slay the Spire 2
```

`STS2 AI Agent` mod をスパイアのゲームディレクトリの `mods/` 以下にコピーします。

Slay the Spire 2 ディレクトリに mods フォルダーがない場合は、自分で作成してください。

```text
mod の使用はセーブデータの紛失を引き起こす可能性があります。バックアップを取るか、コンソールでリカバリーしてください（スパイアのメインメニューで "~" キーを押し、"unlock all" と入力すると、すべてのキャラクターと難易度をアンロックできます）。
```

インストール完了後のディレクトリは以下のようになるはずです：

```text
Slay the Spire 2/
  mods/
    STS2AIAgent.dll
    STS2AIAgent.pck
    mod_id.json
```

### ゲームを起動してインターフェイスを確認

まず通常通りゲームを起動し、Mod がゲームと一緒に読み込まれるようにします。

mod が読み込まれた後、NEKO で猫の手を有効にし、プラグインをオンにし、プラグインパネルに入って、手動で殺戮の塔プラグインを起動します。

### 使用可能なコマンド

【カードを出して】【自動代打】【クリアして】【プレイの調子は？】【停止】
【1 枚カードを出して】【あるカードを出して】【1 枚おすすめして】……などなど…

## 機能概要

- ローカルの `STS2 AI Agent` HTTP サービスに接続し、ゲーム状態を読み取り。
- 手動 1 ステップ実行、バックグラウンド半自動プレイ、一時停止、再開、停止をサポート。
- 3 つの意思決定モード：`full-program`、`half-program`、`full-model` をサポート。
- キャラクターごとの戦略文書の読み込みをサポート。戦略ファイルは `strategies/` にあります。
- ねこ娘の単一カード選択をサポート：現在出せる `play_card` アクションから 1 枚のみ選択し、まず理由をプッシュしてから実行。
- ねこ娘ソフトガイダンスをサポート：ユーザーまたはねこ娘が自然言語でガイダンスを送信でき、次のラウンドの LLM 意思決定が参照します。
- バックグラウンド観察報告をサポート：現在の階層、戦闘、手札、敵の意図、LLM の理由などをフロントエンドにプッシュ。
- 安全保護をサポート：低 HP 一時停止、ボス/危険な攻撃時の減速、HP 回復後の自動復帰、瀕死サバイバル戦略、利益最大化と連携スコアリング。

## 依存関係

このプラグインは、上流の Mod `STS2 AI Agent` が提供するローカル HTTP サービスに依存します：

- ゲーム内 Mod：`STS2AIAgent`
- デフォルトのローカルインターフェイスアドレス：`http://127.0.0.1:8080`
- ヘルスチェックアドレス：`http://127.0.0.1:8080/health`

つまり、このプラグインが動作する前提は：

1. 『Slay the Spire 2』に `STS2 AI Agent` の Mod がインストールされていること。
2. ゲーム起動後、`http://127.0.0.1:8080/health` にアクセスできること。
3. N.E.K.O で `sts2_autoplay` プラグインが有効になっていること。

## 本プラグインの設定

設定ファイル：`plugin.toml`

### 基本設定

| 設定項目 | デフォルト値 | 説明 |
| --- | --- | --- |
| `base_url` | `http://127.0.0.1:8080` | スパイアのローカル Agent アドレス。 |
| `connect_timeout_seconds` | `5` | 接続タイムアウト秒数。 |
| `request_timeout_seconds` | `15` | リクエストタイムアウト秒数。 |
| `poll_interval_idle_seconds` | `3` | アイドル状態のポーリング間隔。 |
| `poll_interval_active_seconds` | `1` | 自動プレイ実行時のポーリング間隔。 |
| `action_interval_seconds` | `1.5` | 各アクション間の追加間隔。 |
| `post_action_delay_seconds` | `0.5` | アクション実行後に盤面が安定するのを待つ間隔。 |
| `autoplay_on_start` | `false` | プラグイン起動後に自動でプレイを開始するか。 |
| `semi_auto_autoplay` | `true` | 自動プレイ起動時に半自動タスクコンテキストを作成するか。 |
| `mode` | `half-program` | 現在の自動プレイモード。 |
| `character_strategy` | `defect` | キャラクター戦略名。`strategies/<name>.md` に対応。 |
| `max_consecutive_errors` | `3` | 最大連続エラー回数。これを超えると切断とみなす。 |
| `push_notifications` | `true` | 履歴保留フィールド。 |
| `event_stream_enabled` | `false` | 予約フィールド、現在は実際には有効になっていません。 |

### 意思決定モード

`mode` は以下の値をサポートし、対応する中国語のエイリアスもサポートします：

| モード | 中国語エイリアス | 説明 |
| --- | --- | --- |
| `full-program` | `全程序` | 純粋なプログラムヒューリスティック、モデル呼び出しなし。 |
| `half-program` | `半程序` | まずプログラム的に事前チェックを行い、次にモデル意思決定を 1 回呼び出し、合法性検証/フォールバックを行う。 |
| `full-model` | `全模型` | 2 回のモデル呼び出し：最初に reasoning、次に final action。途中でプログラムチェック、最後に合法性検証。 |

### キャラクター戦略

`character_strategy` は `strategies/<name>.md` に従って戦略文書を検索します。現在組み込みの戦略：

- `defect`
- `ironclad`
- `silent_hunter`
- `necrobinder`
- `regent`

`strategies/` に Markdown ファイルを追加して戦略を拡張できます。例：

```text
strategies/my_strategy.md
```

そして、設定またはエントリーパラメータを以下のように設定します：

```text
my_strategy
```

### フロントエンドプッシュとねこ娘観察

| 設定項目 | デフォルト値 | 説明 |
| --- | --- | --- |
| `llm_frontend_output_enabled` | `true` | 自動プレイのアクション/エラーをフロントエンドに能動的にプッシュするか。 |
| `llm_frontend_output_probability` | `0.15` | 通常アクションのプッシュ確率。範囲は `0.0 ~ 1.0` に収束します。エラーは強制プッシュ。 |
| `neko_reporting_enabled` | `true` | ねこ娘観察報告をプッシュするか。 |
| `neko_report_interval_steps` | `1` | 何ステップごとに観察報告をプッシュするか。最小 `1`。 |
| `neko_commentary_enabled` | `true` | 観察報告内でねこ娘のリアルタイム実況を生成するか。無効化すると、構造化された観察報告は引き続きプッシュされますが、`live_commentary.text` は空のままとなります。 |
| `neko_commentary_probability` | `0.65` | 通常の低優先度実況のトリガー確率。範囲は `0.0 ~ 1.0` に収束。低 HP、致命的、高攻撃などの高優先度シナリオは確率を回避できます。 |
| `neko_commentary_min_interval_seconds` | `4` | 同じ低優先度シナリオで実況を繰り返す最小間隔秒数。スパムや繰り返しを減らすために使用。 |
| `neko_critical_commentary_always` | `true` | `critical` / `high` 緊急度の実況を常に発するか。例：瀕死、致命的、敵の高攻撃。 |
| `neko_guidance_max_queue` | `50` | ねこ娘ソフトガイダンスキューの最大長。 |

ねこ娘観察報告は、簡略化された `report`、`neko_context`、`live_commentary`、`task` などの metadata を含み、フロントエンドや対話ロジックがこれを「プロセス観察」と判断するのに役立ち、タスク完了通知ではありません。ユーザーのトークンを節約するため、プッシュ内容は現在のアクション、HP、手札、敵、戦術概要、消費されたガイダンス、タスク概要のみを保持します。

`live_commentary` はフロントエンド/TTS に短い口頭説明フィールドを提供します：`text`、`scene`、`mood`、`urgency`、`priority`、`tts`、`interrupt`、`tone`、`character_strategy`。実況はシーンごとにテンプレートプールからランダムに選択されて繰り返しを減らし、キャラクター戦略に応じて傾向を調整します。例：`defect` は理性的、`ironclad` は安定志向。現在カバーしているのは瀕死、低 HP、致命的、敵襲、防御、通常戦闘、報酬、ショップ、休憩所、イベント、マップ、および戦闘終了、重要レリック、ルート選択完了などのイベントレベルの実況です。

### 安全保護と自律的アクション

| 設定項目 | デフォルト値 | 説明 |
| --- | --- | --- |
| `neko_auto_low_hp_threshold` | `0.3` | 現在の HP 比率がこの値を下回ると、バックグラウンド自動プレイは自律的に一時停止します。 |
| `neko_auto_safe_hp_threshold` | `0.5` | HP がこの比率まで回復すると、自動復帰可能。 |
| `neko_auto_dangerous_attack_threshold` | `20` | 敵襲ダメージがこの値に達し、防御を破る場合、自動的に減速。 |
| `neko_auto_resume_after_low_hp` | `true` | 低 HP 一時停止後、HP 回復時の自動復帰を許可するか。 |
| `neko_desperate_enabled` | `true` | 瀕死サバイバル戦略を有効にするか。 |
| `neko_desperate_hp_threshold` | `0.2` | 瀕死サバイバル戦略をトリガーする HP 比率。 |
| `neko_maximize_enabled` | `true` | 利益最大化のカード選択を有効にするか。 |
| `neko_synergy_enabled` | `true` | 連携/シナジースコアリングを有効にするか。 |

現在の自律的アクションには以下が含まれます：

- `pause`：低 HP 時に一時停止し、ユーザーまたはねこ娘の指示を待つ。
- `slow_down`：ボス戦や危険な攻撃時にアクション間隔を一時的に遅くする。
- `resume`：安全 HP 条件を満たした後に復帰。

## 一般ユーザー向け推奨フレーズ

一般ユーザーは下記の低レベルエントリーを覚える必要はありません。優先的にユーザーの原文を `sts2_neko_command` に渡し、プラグイン内部で状態確認、提案提供、実際の出カード、1 ステップ実行、自動プレイ開始、一時停止、再開、停止、最近の出カードのレビュー、自動プレイの質問への回答、または自動プレイ中のソフトガイダンスとして使用するかを判断します。

推奨インタラクションルール：

| ユーザーフレーズ | プラグインの動作 |
| --- | --- |
| `スパイアつながった？` / `今どんな状況？` | 接続、状態、スナップショットの確認のみ。ゲームを操作しない。 |
| `今ターンどう打つ？` / `どのカードがいい？` | 出せるカードを 1 枚だけ推奨し、理由を説明。自動で出カードしない。 |
| `1 枚カードを出して` / `1 枚選んで出して` | 明確な許可後、`play_card` アクションから 1 枚だけ選び、出す。 |
| `1 ステップ打って` / `1 ステップ実行` | 明確な許可後、合法な 1 アクションを実行。ターン終了、報酬選択、マップ移動を含む可能性あり。 |
| `この階を打って` / `自動で打って` | 半自動プレイを開始。デフォルトで現在の階層完了を停止条件とする。 |
| `先に防御して` / `攻撃を欲張らないで` | 自動プレイ実行中はソフトガイダンスとして次のラウンドの意思決定に入る。実行されていない場合は保守的に明確化を求め、勝手に実行しない。 |
| `さっきの打ち方どうだった？` / `さっきのカードを振り返って` | 最近の軽量スナップショットに基づいてプレイ感のレビューを提供。ゲームを操作しない。 |
| `なぜそう打つの？` / `何してるの？` | 自動プレイ実行中、現在の戦略と盤面の根拠に答える。追加操作は行わない。 |
| `ちょっと一時停止` / `続けて` / `止めて` | それぞれ自動プレイを一時停止、再開、または停止。 |

安全デフォルト：相談は操作しない、曖昧な表現では危険なアクションを実行しない。ユーザーが明示的に「打って」「実行」「自動プレイ」「お任せ」と言ったときのみ実際に操作します。

## プラグインエントリー

下記のエントリーはホストに公開されており、N.E.K.O で直接呼び出せます。一般ユーザーシナリオでは `sts2_neko_command` の優先呼び出しを推奨します。その他のエントリーは主に開発者向けの精密制御インターフェイスです。

### `sts2_neko_command`

殺戮の塔自然言語マスターエントリー。ユーザーが低レベルツールを明示的に指定していない場合、優先的に呼び出します。

パラメータ：

- `command`：必須、ユーザーの原文。例：`今ターンどう打つ？`、`1 枚カードを出して`、`先に防御して`、`ちょっと一時停止`。
- `scope`：オプション、デフォルト `auto`。可能な値：`auto`、`status`、`advice`、`one_card`、`one_action`、`autoplay`、`control`、`guidance`、`review`、`question`、`chat`。
- `confirm`：オプション、デフォルト `false`。継続的な「お任せ」などの高リスク操作の確認に使用します。

返り値には `intent`、`action`、`executed`、`needs_confirmation`、`summary`、および基底の `result` が含まれます。

### `sts2_health_check`

ローカルのスパイア Agent サービスが利用可能か確認します。

### `sts2_refresh_state`

現在のスパイア状態を強制的に 1 回リフレッシュします。

### `sts2_get_status`

接続状態、自動プレイ状態、現在のモード、キャラクター戦略、半自動タスク、最近のエラー、最近のアクションなどの情報を取得します。

### `sts2_get_snapshot`

最近キャッシュされたゲームスナップショットと現在実行可能なアクションを取得します。

### `sts2_step_once`

現在の戦略に従って 1 ステップ実行します。

### `sts2_play_one_card_by_neko`

ねこ娘にカードを選ばせて出させます。

パラメータ：

- `objective`：オプション、ユーザー認可目標。例：`1 枚選んで出して`。

動作：

1. 現在のプレイヤー、手札、敵、合法アクションを読み取り。
2. `play_card` アクションのみ保持。
3. 現在のモード/戦略にカードを 1 枚選ばせる。
4. まずフロントエンドに「どのカードを出す予定か、その理由」をプッシュ。
5. アクションが依然として合法か再検証。
6. 出カードを実行し、完了観察をプッシュ。

現在出せるカードがない場合は `idle` を返し、失敗理由をプッシュします。

### `sts2_start_autoplay`

バックグラウンド半自動プレイループを起動します。

パラメータ：

- `objective`：オプション、ユーザー認可目標。例：`この階を打って`。
- `stop_condition`：停止条件、デフォルト `current_floor`。

`stop_condition` がサポート：

- `current_floor`：現在の階層が完了したか、次の階層に入った後に終了。
- `current_combat` / `combat`：タスク中に戦闘に入り、その後戦闘から離脱した後に終了。
- `manual` / `none`：自動完了せず、手動停止が必要。

起動後、プラグインは半自動タスクコンテキストを作成し、タスク開始イベントをフロントエンドにプッシュします。タスク完了時には `semi_auto_task_completed` がプッシュされます。

### `sts2_pause_autoplay`

自動プレイを一時停止します。

### `sts2_resume_autoplay`

一時停止されておりバックグラウンドタスクが依然として存在する自動プレイを再開します。バックグラウンドタスクが既に存在しない場合、安全に `idle` を返し、暗黙的に自動プレイを再起動しません。

### `sts2_stop_autoplay`

自動プレイを停止し、半自動タスクコンテキストをクリアします。

### `sts2_get_history`

最近のアクションと状態履歴を取得します。

パラメータ：

- `limit`：返す件数、デフォルト `20`、範囲は `1 ~ 100` に制限されます。

### `sts2_send_neko_guidance`

バックグラウンド自動プレイにねこ娘ソフトガイダンスを送信します。ガイダンスはキューに入り、次のラウンドの LLM 意思決定時にコンテキストに注入されます。

パラメータ：

- `content`：必須、自然言語のガイダンス内容。例：`先に防御して、攻撃を急がないで`。
- `step`：オプション、対応するステップ数。
- `type`：オプション、デフォルト `soft_guidance`。

### `sts2_set_mode`

自動プレイモードを設定します。

パラメータ：

- `mode`：`full-program` / `全程序`、`half-program` / `半程序`、`full-model` / `全模型` をサポート。

### `sts2_set_character_strategy`

キャラクター戦略名を設定します。

パラメータ：

- `character_strategy`：名前正規化後に `strategies/<name>.md` にマッチします。例：`defect` は `strategies/defect.md` にマッチ。

### `sts2_set_speed`

速度パラメータを設定し、ローカルの `plugin.toml` に書き戻します。

パラメータ：

- `action_interval_seconds`
- `post_action_delay_seconds`
- `poll_interval_active_seconds`

## 典型的な使用方法

### 接続確認

1. 『Slay the Spire 2』を起動。
2. `http://127.0.0.1:8080/health` にアクセスできることを確認。
3. N.E.K.O で `sts2_health_check` を呼び出し。

### 手動 1 ステップ実行

呼び出し：

```text
sts2_step_once
```

プラグインは現在の `mode` と `character_strategy` に基づいて合法なアクションを 1 つ選択して実行します。

### ねこ娘にカードを 1 枚出させる

ユーザーはねこ娘に次のように言うことができます：

```text
1 枚選んで出して
```

ホストは以下を呼び出すべきです：

```text
sts2_play_one_card_by_neko
```

プラグインは現在出せるカードからのみ選び、ターン終了、マップ、報酬、その他のアクションは選びません。

### ねこ娘に階をクリアしてもらう

ユーザーは次のように言うことができます：

```text
この階を打って
```

ホストは以下を呼び出すべきです：

```text
sts2_start_autoplay
```

推奨パラメータ：

```json
{
  "objective": "この階を打って",
  "stop_condition": "current_floor"
}
```

タスク実行中、観察イベントは単なる進捗報告であり、完了を表すものではありません。半自動タスク完了イベントを受け取って初めて、ユーザーにこの階が完了したと伝えるべきです。

### 途中ガイダンス

自動プレイ中、ユーザーまたはねこ娘はガイダンスを送信できます：

```text
先に防御して、ダメージを受けすぎないで
```

以下を呼び出すべきです：

```text
sts2_send_neko_guidance
```

推奨パラメータ：

```json
{
  "content": "先に防御して、ダメージを受けすぎないで",
  "type": "soft_guidance"
}
```

ガイダンスは次のラウンドの LLM 意思決定時に参照されます。`full-program` モードはモデルに依存しないため、ソフトガイダンスの影響は限定的です。

## フロントエンドプッシュイベント

プラグインはホストのメッセージチャネルを通じて以下のカテゴリのイベントをプッシュします。タスク開始/完了、エラー、単一カード予告以外、通常の観察はユーザーのトークン消費を減らすため、できるだけ短いテキストと簡略化された metadata を使用します。

| イベントタイプ | 説明 |
| --- | --- |
| `action` | 通常自動プレイアクション観察、確率制御。 |
| `error` | 自動プレイエラー、強制プッシュ。 |
| `neko_report` | 完全なねこ娘観察報告、現在の盤面、手札、敵、戦術概要、モデルの理由を含む。 |
| `neko_card_task_planned` | ねこ娘単一カードタスクが特定のカードを出す計画。 |
| `neko_card_task_completed` | ねこ娘単一カードタスク実行済み。 |
| `neko_card_task_failed` | ねこ娘単一カードタスク実行不可。 |
| `semi_auto_task_started` | 半自動タスク開始。 |
| `semi_auto_task_completed` | 半自動タスク完了。 |
| `neko_autonomous_action` | システム自律的一時停止、減速、または再開。 |

注意：`neko_report` はプロセス観察であり、タスク完了通知ではありません。フロントエンドや対話ロジックは、単一ステップアクション、出カード、ターン終了、状態リフレッシュを「タスク完了」「ボス撃破」「戦闘終了」「クリア」と表現すべきではありません。ねこ娘が次のラウンドの意思決定に影響を与えたい場合は `sts2_send_neko_guidance` を呼び出すべきです。フローを強制制御したい場合は、一時停止、再開、または停止のエントリーを呼び出します。

## よくあるトラブルシューティング

### プラグインエントリー呼び出し時に接続失敗を報告

最初に確認：

- ゲームが起動しているか。
- `STS2 AI Agent` Mod が正しくゲームの `mods/` に配置されているか。
- `http://127.0.0.1:8080/health` にアクセスできるか。
- `plugin.toml` の `base_url` が正しいか。

### `http://127.0.0.1:8080/health` が開かない

優先的に確認：

1. ゲームが本当に起動しているか。
2. `STS2AIAgent.dll`、`STS2AIAgent.pck`、`mod_id.json` がすべてゲームディレクトリ `mods/` にコピーされているか。
3. ファイル名がシステムにより改名、重複、誤った場所に置かれていないか。
4. 操作しているのが Steam ゲームディレクトリで、上流リポジトリディレクトリではないか。
5. ファイアウォールやセキュリティソフトがローカルポートをブロックしていないか。

### 自動プレイは実行できるが、フロントエンドがメッセージを受け取らない

確認：

- `llm_frontend_output_enabled` が `true` か。
- `llm_frontend_output_probability` が低すぎないか。
- `neko_reporting_enabled` が `true` か。
- デバッグ時に、まず `llm_frontend_output_probability` を `1` に設定してみる。
- ホストのフロントエンドがプラグインのプッシュメッセージを受け取っているか。

### ねこ娘の途中ガイダンスに明らかな効果がない

確認：

- 現在のモードが `half-program` または `full-model` か。
- `sts2_send_neko_guidance` が `ok` を返すか。
- ガイダンスの内容が十分に具体的か。例：「防御優先」「最低 HP の敵を先に攻撃」「ポーション温存」。
- 現在の合法アクションが本当にガイダンスを満たせるか。

### 半自動タスクがなかなか完了しない

`stop_condition` を確認：

- `manual` / `none` の場合、タスクは自動完了せず、`sts2_stop_autoplay` を呼び出す必要があります。
- `current_combat` の場合、タスク中に戦闘に入り、その後戦闘から離脱した後に完了します。
- `current_floor` の場合、通常は現在の階層が完了したか、次の階層に入った後に完了します。

`sts2_get_status` を呼び出して `autoplay.task` を確認できます。

### イベントルーム、ポップアップ、または遷移状態でスタック

現在のバージョンでは、イベント、ポップアップ、遷移状態の処理は既に行われており、優先アクションには以下が含まれます：

- `confirm_modal`
- `dismiss_modal`
- `choose_event_option`
- `proceed`

それでもスタックする場合は、まず `sts2_get_snapshot` で現在の `screen` と `available_actions` を確認します。

### 自動プレイが突然一時停止または減速

安全保護がトリガーされた可能性があります：

- HP 比率が `neko_auto_low_hp_threshold` を下回ると一時停止。
- ボス戦や危険な攻撃時に減速。
- `neko_auto_resume_after_low_hp` が `true` の場合、HP が `neko_auto_safe_hp_threshold` まで回復すると自動復帰する可能性があります。

`sts2_get_status` を呼び出して状態を確認するか、`sts2_resume_autoplay` / `sts2_stop_autoplay` を呼び出して処理できます。
