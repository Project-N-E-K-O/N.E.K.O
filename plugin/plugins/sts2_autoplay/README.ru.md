# Быстрый старт

`sts2_autoplay` используется для подключения к N.E.K.O локального состояния *Slay the Spire 2*, предоставляемого `STS2 AI Agent`. Плагин может читать состояние игры, выполнять допустимые действия, автоматически играть согласно стратегии, позволять неко выбирать одну карту, отправлять наблюдения во фронтенд и позволять неко в фоновых задачах отправлять мягкие подсказки, влияющие на следующий раунд принятия решений.

## Руководство по использованию

### Получение MOD

Используйте Git Clone:
```text
git clone https://gitclone.com/github.com/CharTyr/STS2-Agent.git
```

### Установка игрового мода

В Steam щёлкните правой кнопкой по *Slay the Spire 2*, выберите «Управление -> Просмотреть локальные файлы».

Стандартная директория игры в Steam обычно похожа на:

```text
...\Steam\steamapps\common\Slay the Spire 2
```

Скопируйте мод `STS2 AI Agent` в папку `mods/` директории Slay the Spire 2.

Если в директории Slay the Spire 2 нет папки `mods`, создайте её самостоятельно.

```text
Использование мода может привести к потере сохранений. Сделайте резервную копию или восстановите через консоль (в главном меню Slay the Spire нажмите клавишу "~" и введите "unlock all", чтобы разблокировать всех персонажей и сложности).
```

После установки директория должна выглядеть так:

```text
Slay the Spire 2/
  mods/
    STS2AIAgent.dll
    STS2AIAgent.pck
    mod_id.json
```

### Запуск игры и проверка интерфейса

Сначала запустите игру обычным образом, чтобы мод загрузился вместе с игрой.

После загрузки мода в NEKO включите кошачью лапу, активируйте плагин, войдите в панель плагинов и вручную запустите плагин Slay the Spire.

### Доступные команды

«сыграй карту» / «автопрохождение» / «пройди забег» / «как я играю?» / «стоп»
«сыграй одну карту» / «сыграй конкретную карту» / «порекомендуй карту» ... и тому подобное...

## Обзор функций

- Подключение к локальной HTTP-службе `STS2 AI Agent` и чтение состояния игры.
- Поддержка ручного выполнения одного шага, фонового полуавтоматического прохождения, паузы, возобновления и остановки.
- Поддержка трёх режимов принятия решений: `full-program`, `half-program`, `full-model`.
- Поддержка загрузки документов стратегий по персонажам; файлы стратегий находятся в `strategies/`.
- Поддержка выбора одной карты неко: выбор только из доступных действий `play_card`, сначала отправка обоснования, затем выполнение.
- Поддержка мягких подсказок неко: пользователь или неко могут отправлять подсказки на естественном языке, которые учитываются в следующем раунде LLM-решений.
- Поддержка фоновых отчётов наблюдения: отправка во фронтенд текущего этажа, боя, руки, намерений врагов, обоснований LLM и т. п.
- Поддержка защиты безопасности: пауза при низком HP, замедление при босс-боях/опасных атаках, автовосстановление после восстановления HP, стратегия выживания на грани смерти, максимизация эффективности и оценка синергии.

## Зависимости

Этот плагин зависит от локальной HTTP-службы, предоставляемой апстрим-модом `STS2 AI Agent`:

- Внутриигровой Mod: `STS2AIAgent`
- Адрес локального интерфейса по умолчанию: `http://127.0.0.1:8080`
- Адрес проверки работоспособности: `http://127.0.0.1:8080/health`

То есть для работы этого плагина необходимо:

1. Чтобы мод `STS2 AI Agent` был установлен в *Slay the Spire 2*.
2. После запуска игры `http://127.0.0.1:8080/health` должен быть доступен.
3. В N.E.K.O должен быть включён плагин `sts2_autoplay`.

## Конфигурация плагина

Файл конфигурации: `plugin.toml`

### Базовая конфигурация

| Параметр | По умолчанию | Описание |
| --- | --- | --- |
| `base_url` | `http://127.0.0.1:8080` | Адрес локального Spire Agent. |
| `connect_timeout_seconds` | `5` | Таймаут подключения в секундах. |
| `request_timeout_seconds` | `15` | Таймаут запроса в секундах. |
| `poll_interval_idle_seconds` | `3` | Интервал опроса в режиме ожидания. |
| `poll_interval_active_seconds` | `1` | Интервал опроса при автопрохождении. |
| `action_interval_seconds` | `1.5` | Дополнительный интервал между действиями. |
| `post_action_delay_seconds` | `0.5` | Интервал ожидания стабилизации состояния после действия. |
| `autoplay_on_start` | `false` | Запускать ли автоматическую игру после старта плагина. |
| `semi_auto_autoplay` | `true` | Создавать ли контекст полуавтоматической задачи при запуске автоигры. |
| `mode` | `half-program` | Текущий режим автоигры. |
| `character_strategy` | `defect` | Имя стратегии персонажа, соответствует `strategies/<name>.md`. |
| `max_consecutive_errors` | `3` | Максимальное количество последовательных ошибок; превышение считается отключением. |
| `push_notifications` | `true` | Зарезервированное историческое поле. |
| `event_stream_enabled` | `false` | Зарезервированное поле, в настоящее время не активно. |

### Режимы принятия решений

`mode` поддерживает следующие значения, а также соответствующие китайские псевдонимы:

| Режим | Китайский псевдоним | Описание |
| --- | --- | --- |
| `full-program` | `全程序` | Чисто программная эвристика, без вызовов модели. |
| `half-program` | `半程序` | Сначала программная предпроверка, затем один вызов модели для решения, с проверкой допустимости/откатом. |
| `full-model` | `全模型` | Два вызова модели: сначала reasoning, затем final action; программные проверки между ними и финальная проверка допустимости. |

### Стратегии персонажей

`character_strategy` ищет документы стратегий по пути `strategies/<name>.md`. Текущие встроенные стратегии:

- `defect`
- `ironclad`
- `silent_hunter`
- `necrobinder`
- `regent`

Вы можете добавлять Markdown-файлы в `strategies/` для расширения стратегий. Например:

```text
strategies/my_strategy.md
```

Затем установите конфигурацию или параметр входа:

```text
my_strategy
```

### Отправка во фронтенд и наблюдения неко

| Параметр | По умолчанию | Описание |
| --- | --- | --- |
| `llm_frontend_output_enabled` | `true` | Отправлять ли активно действия/ошибки автоигры во фронтенд. |
| `llm_frontend_output_probability` | `0.15` | Вероятность отправки обычных действий, ограничивается диапазоном `0.0 ~ 1.0`. Ошибки отправляются принудительно. |
| `neko_reporting_enabled` | `true` | Отправлять ли отчёты наблюдений неко. |
| `neko_report_interval_steps` | `1` | Через сколько шагов автоигры отправлять отчёт наблюдения; минимум `1`. |
| `neko_commentary_enabled` | `true` | Генерировать ли в отчётах наблюдений комментарии неко в реальном времени. При отключении структурированные отчёты по-прежнему будут отправляться, но `live_commentary.text` останется пустым. |
| `neko_commentary_probability` | `0.65` | Вероятность срабатывания обычных низкоприоритетных комментариев, ограничивается диапазоном `0.0 ~ 1.0`; высокоприоритетные ситуации, такие как низкое HP, добивание, высокая атака, могут обходить вероятность. |
| `neko_commentary_min_interval_seconds` | `4` | Минимальный интервал в секундах для повторного комментария в той же низкоприоритетной ситуации, чтобы уменьшить спам и повторы. |
| `neko_critical_commentary_always` | `true` | Всегда ли озвучивать комментарии срочности `critical` / `high`, например, на грани смерти, добивание, высокая атака врага. |
| `neko_guidance_max_queue` | `50` | Максимальная длина очереди мягких подсказок неко. |

Отчёты наблюдений неко содержат сокращённые `report`, `neko_context`, `live_commentary`, `task` и другие metadata, помогая фронтенду или диалоговой логике распознать это как «наблюдение процесса», а не уведомление о завершении задачи. Чтобы экономить токены пользователя, отправляемое содержимое сохраняет только текущее действие, HP, руку, врагов, тактическое резюме, потреблённые подсказки и резюме задачи.

`live_commentary` предоставляет фронтенду/TTS короткие поля озвучки: `text`, `scene`, `mood`, `urgency`, `priority`, `tts`, `interrupt`, `tone`, `character_strategy`. Комментарии случайным образом выбираются из пула шаблонов по сцене, чтобы уменьшить повторы; также корректируются предпочтения по стратегии персонажа, например, `defect` склоняется к рациональности, `ironclad` к стабильности. В настоящее время охватывает: на грани смерти, низкое HP, добивание, нападение врагов, защиту, обычный бой, награды, магазин, места отдыха, события, карту, а также комментарии уровня событий, такие как окончание боя, ключевые реликвии и завершение выбора маршрута.

### Защита безопасности и автономные действия

| Параметр | По умолчанию | Описание |
| --- | --- | --- |
| `neko_auto_low_hp_threshold` | `0.3` | Когда соотношение текущего HP падает ниже этого значения, фоновая автоигра автономно ставится на паузу. |
| `neko_auto_safe_hp_threshold` | `0.5` | После восстановления HP до этого соотношения возможно автовозобновление. |
| `neko_auto_dangerous_attack_threshold` | `20` | Автоматическое замедление, когда входящий урон врага достигает этого значения и пробивает защиту. |
| `neko_auto_resume_after_low_hp` | `true` | Разрешено ли автовозобновление после паузы из-за низкого HP. |
| `neko_desperate_enabled` | `true` | Включать ли стратегию выживания на грани смерти. |
| `neko_desperate_hp_threshold` | `0.2` | Соотношение HP, запускающее стратегию выживания на грани смерти. |
| `neko_maximize_enabled` | `true` | Включать ли выбор карт с максимизацией выгоды. |
| `neko_synergy_enabled` | `true` | Включать ли оценку синергии/комбо. |

Текущие автономные действия включают:

- `pause`: пауза при низком HP, ожидание указаний пользователя или неко.
- `slow_down`: временное замедление интервала действий во время босс-боёв или опасных атак.
- `resume`: возобновление после выполнения условия безопасного HP.

## Рекомендуемые формулировки для обычных пользователей

Обычным пользователям не нужно запоминать низкоуровневые входы ниже. Предпочтительно передавать оригинальные слова пользователя в `sts2_neko_command`, и плагин внутренне определит, проверять ли статус, давать ли совет, реально играть карту, выполнить шаг, начать автоигру, поставить на паузу, возобновить, остановить, разобрать недавнюю игру, ответить на вопросы об автоигре или использовать формулировку как мягкую подсказку во время автоигры.

Рекомендуемые правила взаимодействия:

| Формулировка пользователя | Поведение плагина |
| --- | --- |
| `Spire подключен?` / `какая сейчас ситуация` | Проверять только подключение, статус или снимок; не управлять игрой. |
| `как ходить в этом ходу` / `какую карту лучше сыграть` | Рекомендовать только одну карту для игры с объяснением; не играть автоматически. |
| `сыграй за меня карту` / `выбери карту и сыграй` | После явного разрешения выбрать только одну карту из действий `play_card` и сыграть её. |
| `сделай за меня шаг` / `выполни шаг` | После явного разрешения выполнить одно допустимое действие, которое может включать завершение хода, выбор награды или ход по карте. |
| `пройди за меня этаж` / `автоиграй немного` | Запустить полуавтоматическую игру, по умолчанию условие остановки — завершение текущего этажа. |
| `сначала защищайся` / `не жадничай по урону` | Во время автоигры это становится мягкой подсказкой для следующего раунда решений; если автоигра не запущена, консервативно запросить уточнение, не действовать. |
| `как я только что сыграл` / `разбери последнюю карту` | Дать оценку по последнему лёгкому снимку; не управлять игрой. |
| `почему так играешь` / `что ты делаешь` | Во время автоигры отвечать о текущей стратегии и обосновании ситуации; не выполнять дополнительных действий. |
| `пауза` / `продолжай` / `останови` | Соответственно поставить автоигру на паузу, возобновить или остановить. |

Безопасные значения по умолчанию: консультация не управляет игрой, размытые формулировки не выполняют опасные действия; реальные действия выполняются только когда пользователь явно говорит «сыграй за меня», «выполни», «автоиграй», «возьми на себя».

## Точки входа плагина

Следующие точки входа открыты для хоста и могут вызываться напрямую в N.E.K.O. Для сценариев обычных пользователей предпочтительно вызывать `sts2_neko_command`; другие точки входа в основном служат интерфейсами точного контроля для разработчиков.

### `sts2_neko_command`

Главная естественно-языковая точка входа для Slay the Spire. Если пользователь явно не указал низкоуровневый инструмент, предпочтительно вызывать её.

Параметры:

- `command`: обязательный, оригинальные слова пользователя. Например: `как ходить в этом ходу`, `сыграй за меня карту`, `сначала защищайся`, `пауза`.
- `scope`: необязательный, по умолчанию `auto`. Возможные значения: `auto`, `status`, `advice`, `one_card`, `one_action`, `autoplay`, `control`, `guidance`, `review`, `question`, `chat`.
- `confirm`: необязательный, по умолчанию `false`. Используется для подтверждения высокорисковых операций, таких как продолжительная передача управления.

Возврат содержит `intent`, `action`, `executed`, `needs_confirmation`, `summary` и нижестоящий `result`.

### `sts2_health_check`

Проверка доступности локального сервиса Spire Agent.

### `sts2_refresh_state`

Принудительное однократное обновление текущего состояния Spire.

### `sts2_get_status`

Получение информации о состоянии подключения, статусе автоигры, текущем режиме, стратегии персонажа, полуавтоматической задаче, последних ошибках, последних действиях и т. д.

### `sts2_get_snapshot`

Получение последнего кешированного снимка игры и текущих доступных действий.

### `sts2_step_once`

Выполнение одного шага по текущей стратегии.

### `sts2_play_one_card_by_neko`

Позволяет неко выбрать и сыграть карту.

Параметры:

- `objective`: необязательный, цель авторизации пользователя. Например: `выбери карту и сыграй за меня`.

Поведение:

1. Считывает текущего игрока, руку, врагов и допустимые действия.
2. Сохраняет только действия `play_card`.
3. Позволяет текущему режиму/стратегии выбрать карту.
4. Сначала отправляет во фронтенд «какую карту собирается сыграть и почему».
5. Повторно проверяет, что действие всё ещё допустимо.
6. Играет карту и отправляет наблюдение о завершении.

Если в данный момент нет карт для игры, возвращает `idle` и отправляет причину неудачи.

### `sts2_start_autoplay`

Запускает фоновый цикл полуавтоматической игры.

Параметры:

- `objective`: необязательный, цель авторизации пользователя. Например: `пройди за меня этаж`.
- `stop_condition`: условие остановки, по умолчанию `current_floor`.

`stop_condition` поддерживает:

- `current_floor`: завершается после прохождения текущего этажа или перехода на следующий.
- `current_combat` / `combat`: завершается после того, как во время задачи был вход в бой и затем выход из него.
- `manual` / `none`: не завершается автоматически, требует ручной остановки.

После запуска плагин создаёт контекст полуавтоматической задачи и отправляет во фронтенд событие старта задачи. По завершении задачи отправляется `semi_auto_task_completed`.

### `sts2_pause_autoplay`

Ставит автоигру на паузу.

### `sts2_resume_autoplay`

Возобновляет автоигру, поставленную на паузу, фоновая задача которой ещё существует. Если фоновой задачи больше нет, безопасно возвращает `idle` и не перезапускает автоигру неявно.

### `sts2_stop_autoplay`

Останавливает автоигру и очищает контекст полуавтоматической задачи.

### `sts2_get_history`

Получение последней истории действий и состояний.

Параметры:

- `limit`: количество возвращаемых записей, по умолчанию `20`, диапазон ограничивается `1 ~ 100`.

### `sts2_send_neko_guidance`

Отправляет фоновой автоигре мягкую подсказку неко. Подсказка попадает в очередь и инжектируется в контекст в следующем раунде LLM-решений.

Параметры:

- `content`: обязательный, содержимое подсказки на естественном языке. Например: `сначала защищайся, не торопись с уроном`.
- `step`: необязательный, соответствующий номер шага.
- `type`: необязательный, по умолчанию `soft_guidance`.

### `sts2_set_mode`

Устанавливает режим автоигры.

Параметры:

- `mode`: поддерживает `full-program` / `全程序`, `half-program` / `半程序`, `full-model` / `全模型`.

### `sts2_set_character_strategy`

Устанавливает имя стратегии персонажа.

Параметры:

- `character_strategy`: после нормализации имени сопоставляется с `strategies/<name>.md`. Например, `defect` сопоставляется с `strategies/defect.md`.

### `sts2_set_speed`

Устанавливает параметры скорости и записывает их обратно в локальный `plugin.toml`.

Параметры:

- `action_interval_seconds`
- `post_action_delay_seconds`
- `poll_interval_active_seconds`

## Типичные сценарии использования

### Проверка подключения

1. Запустите *Slay the Spire 2*.
2. Убедитесь, что `http://127.0.0.1:8080/health` доступен.
3. В N.E.K.O вызовите `sts2_health_check`.

### Ручное выполнение одного шага

Вызов:

```text
sts2_step_once
```

Плагин выберет и выполнит одно допустимое действие на основе текущих `mode` и `character_strategy`.

### Дать неко сыграть карту

Пользователь может сказать неко что-то вроде:

```text
выбери карту и сыграй за меня
```

Хост должен вызвать:

```text
sts2_play_one_card_by_neko
```

Плагин выбирает только из доступных в данный момент карт и не выбирает завершение хода, карту, награду или другие действия.

### Дать неко пройти этаж

Пользователь может сказать:

```text
пройди за меня этот этаж
```

Хост должен вызвать:

```text
sts2_start_autoplay
```

Рекомендуемые параметры:

```json
{
  "objective": "пройди за меня этот этаж",
  "stop_condition": "current_floor"
}
```

Во время выполнения задачи события наблюдения — это лишь отчёты о ходе работы и не означают завершения. Только при получении события завершения полуавтоматической задачи следует сообщить пользователю, что этаж пройден.

### Подсказки в процессе

Во время автоигры пользователь или неко могут отправить подсказку:

```text
сначала защищайся, не получай слишком много урона
```

Следует вызвать:

```text
sts2_send_neko_guidance
```

Рекомендуемые параметры:

```json
{
  "content": "сначала защищайся, не получай слишком много урона",
  "type": "soft_guidance"
}
```

Подсказка будет учтена в следующем раунде LLM-решений. Режим `full-program` не зависит от модели, поэтому влияние мягких подсказок ограничено.

## События, отправляемые во фронтенд

Плагин отправляет следующие категории событий через канал сообщений хоста. За исключением старта/завершения задачи, ошибок и анонсов одиночных карт, обычные наблюдения стараются использовать короткий текст и сокращённые metadata, чтобы уменьшить расход токенов пользователя.

| Тип события | Описание |
| --- | --- |
| `action` | Обычное наблюдение действия автоигры, контролируется вероятностью. |
| `error` | Ошибка автоигры, отправляется принудительно. |
| `neko_report` | Полный отчёт наблюдения неко, включая текущее состояние, руку, врагов, тактическое резюме и обоснование модели. |
| `neko_card_task_planned` | Задача неко по одной карте планирует сыграть определённую карту. |
| `neko_card_task_completed` | Задача неко по одной карте выполнена. |
| `neko_card_task_failed` | Задача неко по одной карте не может быть выполнена. |
| `semi_auto_task_started` | Полуавтоматическая задача запущена. |
| `semi_auto_task_completed` | Полуавтоматическая задача завершена. |
| `neko_autonomous_action` | Система автономно поставила на паузу, замедлила или возобновила. |

Примечание: `neko_report` — это наблюдение процесса, а не уведомление о завершении задачи. Фронтенд или диалоговая логика не должны описывать одиночное действие, игру карты, завершение хода или обновление состояния как «задача завершена», «босс побеждён», «бой окончен» или «забег пройден». Если неко хочет повлиять на следующий раунд решений, следует вызвать `sts2_send_neko_guidance`; если хочет жёстко контролировать процесс, следует вызвать точки входа паузы, возобновления или остановки.

## Распространённое устранение неполадок

### При вызове точки входа плагина сообщается об ошибке подключения

Сначала проверьте:

- Запущена ли игра.
- Правильно ли мод `STS2 AI Agent` помещён в `mods/` игры.
- Доступен ли `http://127.0.0.1:8080/health`.
- Правильно ли значение `base_url` в `plugin.toml`.

### `http://127.0.0.1:8080/health` не открывается

Приоритетно проверьте:

1. Действительно ли игра запущена.
2. Скопированы ли `STS2AIAgent.dll`, `STS2AIAgent.pck`, `mod_id.json` в директорию игры `mods/`.
3. Не были ли имена файлов изменены системой, продублированы или помещены не в ту директорию.
4. Работаете ли вы с директорией Steam-игры, а не с директорией репозитория.
5. Не блокируется ли локальный порт брандмауэром или антивирусом.

### Автоигра работает, но фронтенд не получает сообщений

Проверьте:

- Является ли `llm_frontend_output_enabled` равным `true`.
- Не слишком ли низкое значение `llm_frontend_output_probability`.
- Является ли `neko_reporting_enabled` равным `true`.
- При отладке можно сначала установить `llm_frontend_output_probability` в `1`.
- Получает ли фронтенд хоста сообщения, отправляемые плагином.

### Подсказки неко в процессе не дают заметного эффекта

Проверьте:

- Является ли текущий режим `half-program` или `full-model`.
- Возвращает ли `sts2_send_neko_guidance` значение `ok`.
- Достаточно ли конкретно содержимое подсказки, например: «приоритет защиты», «сначала бить врага с наименьшим HP», «сохранять зелье».
- Могут ли текущие допустимые действия действительно выполнить подсказку.

### Полуавтоматическая задача долго не завершается

Проверьте `stop_condition`:

- Если `manual` / `none`, задача не завершится автоматически, нужно вызвать `sts2_stop_autoplay`.
- Если `current_combat`, задача завершится после того, как во время задачи был вход в бой и затем выход из него.
- Если `current_floor`, обычно завершается после прохождения текущего этажа или перехода на следующий.

Можно вызвать `sts2_get_status` и просмотреть `autoplay.task`.

### Залипание в комнатах событий, всплывающих окнах или переходных состояниях

Текущая версия уже обрабатывает события, всплывающие окна и переходные состояния, приоритетные действия включают:

- `confirm_modal`
- `dismiss_modal`
- `choose_event_option`
- `proceed`

Если по-прежнему залипает, сначала используйте `sts2_get_snapshot` и просмотрите текущие `screen` и `available_actions`.

### Автоигра внезапно ставится на паузу или замедляется

Возможно, сработала защита безопасности:

- Когда соотношение HP падает ниже `neko_auto_low_hp_threshold`, ставится на паузу.
- В босс-боях или при опасных атаках замедляется.
- Если `neko_auto_resume_after_low_hp` равно `true`, после восстановления HP до `neko_auto_safe_hp_threshold` может автоматически возобновиться.

Можно вызвать `sts2_get_status` для проверки состояния или вызвать `sts2_resume_autoplay` / `sts2_stop_autoplay` для обработки.
