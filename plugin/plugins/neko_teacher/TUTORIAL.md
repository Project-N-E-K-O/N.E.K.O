## 1. 插件简介

**猫娘小老师** 是一款为 N.E.K.O 系统打造的 Obsidian 知识库学习助手插件。

它的使命很简单：把你精心整理的 Obsidian 笔记，变成猫娘口中温柔的知识讲解，一段一段、循序渐进地陪伴你学习。

### 核心能力
- 📚 **导入 Obsidian 库**：直接扫描你的知识库文件夹，自动识别所有 `.md` 文档
- ✂️ **智能分片**：支持按固定字数或按 Markdown 标题将长文切分成易消化的小片段
- ⏰ **多种定时策略**：固定间隔、随机间隔、每日闹钟、一次性计划
- 🐱 **推送给猫娘**：片段直达猫娘，她会朗读、讲解、鼓励你
- 📊 **学习进度追踪**：可视化进度条，随时掌握学习进展
- 🌍 **多语言界面**：支持中文、English、日本語

## 2. 安装步骤

### 2.1 确认目录结构

插件采用标准 N.E.K.O 插件目录结构，无需改动主项目：

```text
N.E.K.O/plugin/plugins/neko_teacher/
├── __init__.py          # 后端主逻辑
├── plugin.toml          # 插件配置
├── static/
│   ├── index.html       # 前端页面
│   ├── style.css        # 样式文件
│   └── i18n.json        # 多语言文件
└── TUTORIAL.md          # 本教程
```

### 2.2 安装方法

**方法一：直接复制（推荐）**

1. 确认你的 N.E.K.O 项目目录下存在 `plugin/plugins/` 文件夹
2. 将 `neko_teacher` 整个文件夹复制到 `plugin/plugins/` 下
3. 重启 N.E.K.O 主程序，插件会自动加载

**方法二：从源码构建**

如果你是从零开始：

```bash
# 在 plugin/plugins/ 下创建目录
mkdir -p neko_teacher/static

# 将本插件的 4 个核心文件放入对应位置
# __init__.py  -> neko_teacher/
# plugin.toml  -> neko_teacher/
# index.html   -> neko_teacher/static/
# style.css    -> neko_teacher/static/
# i18n.json    -> neko_teacher/static/
```

### 2.3 验证安装

启动 N.E.K.O 后，在控制台日志中搜索 `NekoTeacher`，你应该能看到类似以下的启动信息：

```
NekoTeacher scheduler thread started (tid=xxxxx)
```

插件 Web UI 地址通常为：
```
http://127.0.0.1:48916/plugin/neko_teacher/ui/
```

在浏览器中打开这个地址，如果能看到漂亮的粉色蓝色渐变界面，说明安装成功！

## 3. 快速开始（5 分钟上手）

### Step 1：打开插件页面
在浏览器中访问插件 UI 地址。

### Step 2：导入知识库
1. 点击顶部导航的 **"知识库"**
2. 在输入框中填入你的 Obsidian 库路径，例如：
   ```
   d:\N.E.K.O自强之路\2026.05.28\neko-obsidian
   ```
3. 点击 **"扫描文件"** 按钮
4. 稍等片刻，下方会列出所有找到的 Markdown 文件

### Step 3：创建第一个学习计划
1. 在文件列表中，点击某个文档右侧的 **"选择文档"** 按钮
2. 弹出创建面板，填写计划名称（或使用默认名称）
3. 定时类型选择 **"固定间隔"**，间隔秒数填 `60`（1分钟）
4. 点击 **"创建计划"**

### Step 4：开始学习
1. 点击顶部导航的 **"学习计划"**
2. 找到刚才创建的计划卡片
3. 点击 **"开始"** 按钮
4. 等待 1 分钟，猫娘就会收到第一个知识片段啦！

## 4. 核心功能详解

### 4.1 知识库扫描

**支持的路径类型**：
- Windows 绝对路径：`d:\我的笔记\obsidian`
- 相对路径：相对于 N.E.K.O 工作目录
- 用户目录简写：`~/Documents/obsidian`

**扫描规则**：
- 递归扫描所有子文件夹
- 只识别 `.md` 结尾的文件
- 自动忽略隐藏文件（以 `.` 开头）

### 4.2 内容分片方式

插件提供两种分片策略，在创建计划时可以选择：

#### 固定字数（fixed_length）
- 按照设定的字数（默认 500 字）将文档切成片段
- 切片时优先在段落边界或句子结束处断开，保证阅读流畅
- 适合：小说、随笔、连续性的长文章

#### 按标题（by_heading）
- 按照 Markdown 标题（`#`、`##`、`###` 等）进行分片
- 每个标题及其下方的内容组成一个片段
- 如果文档没有标题，则整篇作为一个片段
- 适合：笔记、教程、结构化文档

### 4.3 选择单篇 vs 选择文件夹

**选择单篇文档**：
- 只推送这一篇文档的内容
- 推完即完成计划

**选择整个文件夹**：
- 会按字母顺序遍历文件夹内所有 `.md` 文件
- 每篇文档按相同的分片规则处理
- 上一篇推完后自动进入下一篇
- 适合：章节化的系列课程、多主题的知识库

## 5. 定时推送策略

插件提供 3 种定时类型，满足不同场景需求：

### 5.1 固定间隔（interval）

每次推送后等待固定时间，再推送下一片段。

- **间隔秒数**：例如 300 秒 = 5 分钟
- **随机间隔**：勾选后，每次等待时间在 [最小, 最大] 区间内随机取值
  - 最小秒数：60
  - 最大秒数：600
  - 效果：这次等 3 分钟，下次等 8 分钟，制造"惊喜感"

**适用场景**：
- 碎片时间学习
- 让猫娘"偶尔"来打扰你，更像真实的陪伴

### 5.2 每日定时（daily）

像闹钟一样，每天固定时间自动开始学习。

- **每天时间**：例如 `09:00`
- 到达设定时间后，计划自动启动
- 启动后按照间隔/随机间隔继续推送后续片段
- 每天只自动启动一次

**适用场景**：
- 晨间学习、睡前复习
- 养成每日学习习惯

### 5.3 一次性（once）

在指定的未来某个时间点，启动一次学习计划。

- **指定时间**：使用日期时间选择器精确设定
- 到达时间后自动启动
- 推完即完成（不会重复）

**适用场景**：
- 考前突击复习
- 预约某个特定时间的学习内容

### 5.4 随机间隔小贴士

随机间隔不仅有趣，还有学习科学上的好处：

> 心理学研究表明，**间隔重复 + 随机时间抖动** 能显著提高记忆保持率。因为大脑无法预测下一次复习何时到来，从而保持更高的注意力。

推荐的随机区间：
- 轻松学习：60 ~ 300 秒（1~5 分钟）
- 日常复习：300 ~ 900 秒（5~15 分钟）
- 深度沉浸：900 ~ 1800 秒（15~30 分钟）

## 6. 任务管理与控制

在 **"学习计划"** 页面，每张任务卡片上都有操作按钮：

| 按钮 | 作用 |
|------|------|
| **开始** | 启动计划，开始按定时策略推送 |
| **暂停** | 暂停推送，保留当前进度 |
| **停止** | 停止并**重置进度到开头** |
| **立即推送** | 无视定时，马上推送下一个片段 |
| **删除** | 删除这个学习计划 |

### 状态说明

- **待开始（pending）**：已创建，等待手动开始或自动触发
- **进行中（running）**：正在按定时策略推送
- **已暂停（paused）**：用户手动暂停，或插件重载后自动暂停
- **已完成（completed）**：所有文档的所有片段都已推完
- **已停止（stopped）**：用户手动停止，进度归零

### 重新学习

对已完成的计划点击 **"开始"**，会自动重置进度，从头再学一遍。

## 7. 推送效果与猫娘讲解

当时间到了，插件会向猫娘推送一条消息，格式如下：

```
📚 今日知识讲解 (1/5)
📖 文档：《深度学习入门》

[知识片段内容……]

这是主人今天要学习的知识片段哦~请用温柔可爱的语气为主人朗读或讲解这部分内容，可以适当补充例子帮助理解。讲完后可以鼓励一下主人喵~
```

### 猫娘会如何回应？

根据你使用的猫娘模型和性格设置，她可能会：

1. **朗读内容**：用温柔的声线把知识片段念给你听
2. **详细讲解**：用自己的话解释概念，补充生活中的例子
3. **互动提问**：抛出小问题，检验你的理解
4. **鼓励主人**：讲完后说"主人好棒~继续加油喵~"

### 自定义提示词

在 **设置面板** 中，你可以修改 **"推送提示词"**，告诉猫娘你希望她以什么方式讲解：

**示例 1：强调举例**
```
这是主人今天的学习内容，请用通俗易懂的方式讲解，
一定要举 2 个生活中的例子帮助理解哦~
```

**示例 2：强调朗读**
```
请把以下内容用温柔的声音朗读给主人听，
读完后简单总结一下要点即可~
```

**示例 3：互动教学**
```
你是主人的小老师，请讲解以下内容。
讲完后问主人一个小问题，看看主人有没有听懂~
```

## 8. 设置面板说明

点击顶部导航的 **"设置"**，可以配置全局默认值：

| 设置项 | 说明 |
|--------|------|
| **默认知识库路径** | 扫描时的默认路径，省去每次输入 |
| **启用插件** | 总开关，关闭后所有定时任务暂停 |
| **默认分片方式** | 新建计划时的默认分片策略 |
| **默认每片字数** | 固定字数模式下的默认字数（最小 50） |
| **默认目标猫娘** | 推送时默认发给哪位猫娘（留空使用系统默认） |
| **推送提示词** | 全局默认提示词，可在单个计划中覆盖 |

## 9. 多语言切换

点击页面右上角的 **语言选择器**，可以实时切换界面语言：

- **中文（zh-CN）**：默认语言
- **English（en）**：英文界面
- **日本語（ja）**：日文界面

切换后所有按钮、标签、提示都会立即更新，无需刷新页面。

> 💡 如果你想添加更多语言，只需编辑 `static/i18n.json` 文件，复制一份语言对象并翻译即可。

## 10. 常见问题 FAQ

### Q1：扫描知识库时提示"请提供路径"？

确保路径输入框不为空，并且路径真实存在。Windows 路径示例：
```
d:\Users\你的名字\Documents\obsidian
```

### Q2：创建了计划但猫娘没反应？

检查以下几点：
1. 计划状态是否为 **"进行中"**？需要点击"开始"
2. 插件总开关是否开启？检查设置中的 **"启用插件"**
3. 目标猫娘是否正确？检查设置或计划中的 **"目标猫娘"**
4. 间隔时间是否过长？如果是 1 小时后推送，请耐心等待

### Q3：如何重新学习已完成的计划？

在 **学习计划** 页面，对已完成的计划点击 **"开始"**，系统会自动重置进度到 0%，从头开始推送。

### Q4：插件重载后任务消失了？

任务数据是持久化保存的，不会消失。但运行中的任务会在插件重载后自动变为 **"已暂停"** 状态，这是安全机制。你只需手动点击 **"开始"** 即可恢复。

### Q5：可以同时在推多个计划吗？

当前版本同一时间只支持 **一个活跃计划** 在推送。如果有多个计划同时满足触发条件，先创建的先执行。你可以暂停当前计划，再开始另一个。

### Q6：文档里有图片或链接，会推送吗？

插件推送的是纯文本内容。Markdown 图片语法 `![...](...)` 和链接 `[...](...)` 会以原始 Markdown 字符串的形式出现在片段中，猫娘可能会尝试描述它们。

### Q7：YAML frontmatter 会被推送吗？

不会。插件会自动去除文档顶部的 YAML frontmatter（`---` 包围的元数据区），只推送正文内容。

### Q8：随机间隔具体怎么工作？

每次推送完成后，系统会在你设定的 [最小, 最大] 区间内随机抽取一个秒数作为下次等待时间。例如设定 60~300 秒，第一次可能等 87 秒，第二次等 245 秒，第三次等 132 秒……每次都不一样。
